# 原型测试指南

## 如何运行
1. 确保安装Godot 4.6.1 (`D:\3_Tool\Godot_v4.6.1-stable`)
2. 打开Godot，选择"导入项目"
3. 选择 `prototypes/outing-return-cycle/` 目录
4. 点击运行按钮（或按F5）

## 预期行为
1. **窗口出现**：桌面右上角出现320×320透明悬浮窗
2. **初始状态**：状态显示"在家"，外出压力从0开始累积
3. **时间流逝**：每秒 = 1分钟（加速模拟）
4. **循环开始**：
   - 20秒冷却期（无外出）
   - 冷却后开始累积外出压力
   - 达到阈值后触发外出
   - 外出10-45秒后归来
   - 归来后进入20秒冷却期

## 观察要点
### 1. 节奏感
- 循环频率是否合适？（太频繁=打扰，太稀疏=存在感弱）
- 外出时长感觉如何？（太短=匆忙，太长=消失太久）
- 冷却期是否必要？（归来后立即又能出发 vs 需要休息）

### 2. 陪伴感
- 角色离开时，你是否会期待它回来？
- 归来反馈是否让你感到"它有自己的生活"？
- 整体感觉是"陪伴"还是"打扰"？

### 3. 技术表现
- 窗口透明度和置顶是否正确？
- 拖拽功能是否正常？
- 状态显示是否清晰？
- 日志信息是否易读？

### 4. 手动控制
- 测试"手动外出"按钮（在家状态可用）
- 测试"手动归来"按钮（外出状态可用）
- 观察手动干预对循环的影响

### 定性反馈
**节奏感受**（1-5分）：
- 1分：太频繁，感到打扰
- 3分：适中，感觉自然
- 5分：太稀疏，存在感弱

**陪伴感**（1-5分）：
- 1分：感觉像定时器，没有情感连接
- 3分：能感觉到角色有自己的生活
- 5分：强烈期待感和情感连接

**技术满意度**（1-5分）：
- 1分：有明显bug或体验问题
- 3分：基本功能正常
- 5分：流畅稳定，体验良好

**总体推荐**：
- ✅ PROCEED：概念验证成功，可以进入生产实现
- 🔄 PIVOT：需要调整某些方面（请说明）
- ❌ KILL：概念不可行（请说明原因）

## 测试时长建议
- **短测试**：观察3-5个完整循环（约5-10分钟）
- **长测试**：挂机观察30分钟以上，模拟真实使用场景
