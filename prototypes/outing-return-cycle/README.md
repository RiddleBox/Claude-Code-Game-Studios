# 外出-归来循环原型 (Outing-Return Cycle Prototype)

## 核心问题
**外出-归来循环是否能创造基本的"陪伴感"？**

玩家是否能接受角色周期性离开并返回的概念？这个循环是否感觉自然、不打扰，又能创造期待感？

## 原型目标
验证C2外出-归来循环的核心机制可行性，包括：
1. **时间节奏**：保底冷却 → 随机出发 → 外出时长 → 归来反馈
2. **玩家感受**：角色"有自己的生活"的感觉是否自然
3. **技术基础**：透明悬浮窗 + 时间加速模拟的可行性

## 简化假设
- 时间加速：1秒 = 1分钟（便于快速测试循环）
- 状态简化：只有"在家"、"外出中"、"已归来"三个状态
- 反馈简化：文本反馈代替完整叙事碎片
- 窗口简化：基础透明悬浮窗，跳过点击穿透等高级特性

## 技术栈
- **引擎**: Godot 4.6.1 (与主项目保持一致)
- **语言**: GDScript
- **窗口**: 透明、置顶、可拖拽
- **时间**: 加速模拟 (1秒 = 1分钟)

## 预期指标
- **循环节奏**: 20秒冷却 → 随机出发 → 10-45秒外出 → 归来
- **用户反馈**: "感觉自然" vs "感觉打扰" vs "感觉期待"
- **技术可行性**: 窗口系统 + 时间系统能正常协作

## 使用说明
1. 在Godot 4.6.1中打开此目录
2. 运行主场景 `outing_window.tscn`
3. 观察窗口在桌面角落的行为
4. 关注状态变化和反馈文本
5. 调整参数查看节奏变化
