# Fe1对话系统 - 内容生产指南
**版本：** 1.0  
**最后更新：** 2026-04-12  
**适用人群：** 内容策划、文案编辑

---

## 📋 概述
对话系统是角色和用户互动的核心方式，角色会在不同场景下触发不同的对话，显示在头顶的气泡里。本指南说明如何添加、编辑对话内容，不需要编写代码。

---

## 🎯 对话类型
对话分为两大类，分别对应不同的触发方式：

### 1. 空闲对话（idle_dialogues）
角色处于空闲状态时，会随机触发的对话，触发间隔为1~3分钟随机。
适合放一些通用的、自言自语式的日常对话，比如吐槽天气、提醒喝水、无聊的碎碎念等。

### 2. 事件对话（event_dialogues）
特定事件发生时触发的对话，目前支持的事件类型：
| 事件类型 | 触发时机 |
|----------|----------|
| `outing_start` | 角色开始外出时 |
| `outing_return` | 角色外出归来时 |
| `mouse_enter` | 鼠标移动到角色身上时 |
| `mouse_click` | 鼠标点击角色时 |
| `state_idle` | 角色切换到空闲状态时 |
| `state_sleeping` | 角色切换到睡觉状态时 |
| `memory_unlocked` | 解锁了新的回忆碎片时（后续功能） |
| `achievement_unlocked` | 解锁成就时（后续功能） |

## 📝 对话配置格式
所有对话都存储在 `data/config/dialogues/*.json` 文件里，使用标准JSON格式，支持中文。

### 单条对话格式
```json
{
  "content": "对话内容，最多24个字（超出会自动换行）",
  "style": 0,  // 气泡样式，取值参考下方说明
  "weight": 10 // 权重，数值越高被随机选中的概率越大
}
```

### 气泡样式说明（style字段取值）
| 数值 | 样式名称 | 视觉效果 | 适用场景 |
|------|----------|----------|----------|
| 0 | NORMAL | 白色边框+半透明白色背景+白色文字 | 普通对话、日常交流 |
| 1 | THINKING | 灰色边框+半透明灰色背景+浅灰色文字 | 思考、自言自语、疑问 |
| 2 | SURPRISE | 橙色边框+半透明橙色背景+橙色文字 | 惊讶、意外、突然被吓到 |
| 3 | HAPPY | 黄色边框+半透明黄色背景+黄色文字 | 开心、愉快、惊喜、积极情绪 |
| 4 | SAD | 蓝色边框+半透明蓝色背景+蓝色文字 | 难过、委屈、失望、消极情绪 |

### 权重说明（weight字段）
- 权重是0~100之间的整数，数值越高被随机选中的概率越大
- 如果所有对话权重都是10，那么每条概率均等
- 如果A对话权重是20，B是10，那么A的触发概率是B的两倍
- 相同内容的对话不要重复添加，通过调整权重控制出现频率即可

## ✅ 内容规范
1. **长度限制**：单条对话最多24个字，超出会自动换行，建议尽量控制在20字以内更美观
2. **语气要求**：符合角色设定，友好、自然、像真实的朋友对话，避免生硬、机械
3. **合规要求**：不能包含敏感内容、粗口、低俗、政治相关等违规内容
4. **场景匹配**：对话内容要符合对应的事件场景，比如外出归来的对话要和外出相关
5. **避免重复**：相同含义的对话不要添加多条，通过权重调整出现频率即可
6. **通用性**：尽量避免使用时效性强的内容（比如节日、热点事件），除非专门做节日版本

### 正面示例 ✅
```json
{
  "content": "今天天气真好，适合去散步~",
  "style": 3,
  "weight": 10
}
```

### 反面示例 ❌
```json
{
  "content": "今天是2026年4月12日，天气晴朗，万里无云，适合出去散步呼吸新鲜空气", // 太长
  "style": 3,
  "weight": 10
}
```

## 📁 文件配置说明
### 配置文件结构
```
data/config/dialogues/
├─ default_dialogues.json  // 默认对话，基础内容，所有人通用
└─ custom_dialogues.json   // 自定义对话，用户可以自己添加的内容
```
多个配置文件会自动合并，相同事件的对话会叠加。

### 添加新对话的步骤
1. 打开对应的JSON配置文件
2. 找到对应的对话分类（idle_dialogues或者对应的event_dialogues下的事件类型）
3. 在数组里添加符合格式的对话对象，注意JSON语法（最后一条不要加逗号）
4. 保存文件，重启应用即可生效，不需要修改代码

### 示例：给外出事件添加新对话
```json
"outing_start": [
  // 原有对话...
  {
    "content": "我去买杯奶茶，很快回来~",
    "style": 3,
    "weight": 8
  }
]
```

## 🔍 验证与测试
添加新对话后，可以通过以下方式测试是否生效：
1. 启动应用，等待对应事件触发
2. 或者在调试模式下使用命令直接触发对应事件的对话
3. 如果对话没有出现，检查：
   - JSON格式是否正确（有没有漏逗号、引号不配对等）
   - 对话是否加对了分类
   - 权重是不是设太低了
   - 控制台有没有报错信息

---

## 🎨 自定义拓展（可选）
如果需要添加新的事件类型触发对话，可以在`event_dialogues`下添加新的键值对，键名是事件名称，值是对话数组，然后通知开发人员在代码里关联对应的触发时机即可。
