# Fe6 — 通知/提醒系统（Notification System）

> **Status**: In Design
> **Author**: Claude Code Game Studios
> **Last Updated**: 2026-03-30
> **Implements Pillar**: 陪伴不打扰（通知是留痕，不是打断）

---

## Overview

Fe6 通知/提醒系统是窗语游戏中所有「事件留痕」的管理层。它负责在三类事件发生时，以不打断玩家注意力的方式告知玩家：角色外出、角色归来（托盘层）、泄漏内容更新。

Fe6 的核心原则是「通知是留痕，不是打断」——所有通知都是被动可见的，玩家主动注意才会看到，不主动打断玩家的工作流。通知的强度由 C5 性格变量系统通过 `get_notification_intensity()` 接口决定，内敛角色留痕轻微，热烈角色提示明显。

Fe6 的触达范围限定为两层：（1）游戏窗口内的视觉提示；（2）系统托盘图标状态变化。不触发操作系统级 Toast 通知。

Fe6 只负责视觉呈现。对应的音效触发由 Fe5 声音系统负责——Fe6 发出事件信号，Fe5 监听并决定播放内容，Fe6 本身不调用任何音频接口。

## Player Fantasy

你在专心工作。屏幕角落的小窗口安安静静。

某一刻你抬起头瞥了一眼——它不见了，但角落有一张小纸条。你凑近看，它写着它去哪了。你笑了一下，继续工作。

后来你再抬头，它回来了。也许托盘图标亮了一下，也许它就那样静静出现，好像什么都没发生。如果是那种热情的家伙，它可能还会在窗口边上轻轻动一动，像在说「我回来了」——但也只是那一下，不会一直烦你。

你从来不觉得被打扰。你只是，在合适的时候，知道了它在哪里。

## Detailed Design

### Core Rules

**通用规则**

1. Fe6 不主动打断玩家——所有通知均为被动可见，玩家不注意不会有任何弹窗或焦点抢夺。
2. Fe6 只负责视觉提示；音效触发通过信号由 Fe5 处理。
3. 通知强度统一由 C5 的 `get_notification_intensity() → float (0.0–1.0)` 决定，Fe6 不直接读取 C5 原始变量。
4. 所有通知视觉提示在玩家注意到（悬停/点击）后自动消除，不需要显式「已读」操作。
5. 同一事件类型同时触发多次时，合并为一条留痕（不堆叠多个纸条图标）。

**外出通知（纸条）**

6. 角色进入外出态（C2 发出 `character_departed` 信号）时，Fe6 在角色原位置显示「纸条」留痕。
7. 纸条留痕包含一行外出留言文字，内容由 C4 事件线提供；C4 未提供时使用默认文本「出去逛逛，待会回来」。
8. 纸条留痕持续显示直到角色归来，不自动消失。
9. 纸条不触发托盘图标变化（强度无关）。
10. Fe6 记录纸条是否被玩家注意到（悬停或点击），归来时将 `note_unread: bool` 随 `character_returned` 信号一并传递。

**归来通知**

11. 角色归来（C2 发出 `character_returned` 信号）时，Fe6 移除纸条留痕。
12. 归来的窗口内视觉提示由 Fe1 的 RETURN_HINT 负责；Fe6 仅在强度 ≥ 0.3 时额外触发托盘图标变化。
13. 托盘图标变化持续到玩家点击归来提示气泡（Fe1 发出 `return_hint_acknowledged` 信号）后恢复正常。
14. **[Post-MVP]** 若纸条整个外出期间未被注意（`note_unread: true`），Fe6 向 C4 传递该标记；C4 可据此为高热情角色生成「你没注意到我走」类碎片，内敛角色忽略该标记。MVP 阶段 Fe6 传递标记但 C4 不处理。

**泄漏内容通知**

15. Fe2 发出 `leak_content_available` 信号时，Fe6 在窗口内显示轻提示（小图标/角标）。
16. 泄漏内容提示强度 < 0.3 时：仅显示静态小图标；≥ 0.3 时：图标有轻微动画；≥ 0.7 时：额外触发托盘图标变化。
17. 玩家点击泄漏内容区域后，提示自动消除。

### Notification Intensity

C5 通过 `get_notification_intensity() → float (0.0–1.0)` 暴露当前角色的通知强度。Fe6 将其映射为三档：

| 档位 | 范围 | 窗口内行为 | 托盘行为 |
|------|------|-----------|----------|
| 轻微（Subtle） | 0.0–0.3 | 静态纸条/静态小图标，无动画 | 无变化 |
| 温和（Gentle） | 0.3–0.7 | 纸条/图标有轻微动画（淡入淡出/轻微摇摆） | 归来时托盘图标变色 |
| 明显（Expressive） | 0.7–1.0 | 较明显动画，持续时间稍长 | 归来/泄漏内容均触发托盘图标变化 |

**规则**：
- 强度值在角色外出时读取一次，整个外出周期内不变（避免外出途中性格变化导致通知行为中途切换）
- 托盘图标变化统一为图标变色（具体颜色/样式由美术决定），不使用系统闪烁 API

### Event Catalogue

| 事件 | 触发信号 | 来源系统 | Fe6 行为 | 消除条件 |
|------|---------|---------|---------|----------|
| 角色外出 | `character_departed` | C2 | 显示纸条留痕 + 外出留言文字 | 角色归来时自动移除 |
| 角色归来 | `character_returned` | C2 | 移除纸条；强度≥0.3 时触发托盘变色 | Fe1 发出 `return_hint_acknowledged` |
| 泄漏内容更新 | `leak_content_available` | Fe2 | 显示泄漏内容角标；强度≥0.7 时触发托盘变色 | 玩家点击泄漏内容区域 |

## Edge Cases

| # | 场景 | 处理方式 |
|---|------|----------|
| EC-01 | 角色外出期间再次收到 `character_departed` 信号 | 忽略，纸条已存在，不重复显示；留言文字不覆盖（保留首次外出留言） |
| EC-02 | 角色归来信号到达时纸条从未被注意 | 移除纸条，传递 `note_unread: true` 给 C4；MVP 阶段 C4 不处理该标记 |
| EC-03 | 游戏窗口隐藏时角色归来 | 托盘图标变色（强度≥0.3）；窗口重新显示时 Fe1 正常触发 RETURN_HINT |
| EC-04 | 外出期间多次收到 `leak_content_available` | 合并为一个角标，不堆叠；角标消除后若再次收到则重新显示 |
| EC-05 | 强度值在 0.3 / 0.7 临界点附近浮动 | 强度值在外出开始时读取并锁定，整个周期内不重新读取，避免临界抖动 |
| EC-06 | C5 `get_notification_intensity()` 返回异常值（< 0 或 > 1） | 钳制到 [0.0, 1.0] 范围，按 Subtle 档处理 |
| EC-07 | 玩家在纸条显示期间拖动窗口 | 纸条跟随窗口移动，始终显示在角色原位置的相对坐标 |

## Dependencies

**上游依赖（Fe6 依赖这些系统）**

| 系统 | 依赖内容 | 接口 |
|------|---------|------|
| C2 外出-归来循环 | 外出/归来事件触发 | 信号 `character_departed`、`character_returned` |
| C4 事件线系统 | 外出留言文字内容 | 方法 `C4.get_departure_note() → String` |
| C5 性格变量系统 | 通知强度值 | 方法 `C5.get_notification_intensity() → float` |
| Fe1 对话系统 | 归来提示已确认信号 | 信号 `return_hint_acknowledged` |
| Fe2 泄漏内容系统 | 泄漏内容更新事件 | 信号 `leak_content_available` |
| F1 桌面窗口系统 | 托盘图标状态控制 | 方法 `F1.set_tray_state(state: TrayState)` |

**下游依赖（这些系统依赖 Fe6）**

| 系统 | 依赖内容 | 接口 |
|------|---------|------|
| Fe5 声音系统 | 监听 Fe6 事件信号以触发对应音效 | 信号 `notification_shown(event_type)`、`notification_dismissed(event_type)` |
| C4 事件线系统 | 接收纸条未读标记（Post-MVP） | 信号 `note_unread_on_return` |

## Tuning Knobs

| 参数 | 默认值 | 安全范围 | 影响 |
|------|--------|---------|------|
| `subtle_threshold` | 0.3 | 0.1–0.5 | Subtle/Gentle 档分界线；调低让更多角色走轻动画 |
| `expressive_threshold` | 0.7 | 0.5–0.9 | Gentle/Expressive 档分界线；调低让托盘变化更容易触发 |
| `note_animation_duration` | 0.4 秒 | 0.2–1.0 秒 | 纸条/图标动画的持续时长；过长显得拖沓 |
| `tray_color_change_duration` | 持续到消除条件 | — | 托盘变色不自动超时，只由消除条件触发恢复 |

## Acceptance Criteria

| # | 测试条件 | 通过标准 |
|---|---------|----------|
| AC-01 | C2 发出 `character_departed`，角色强度 < 0.3 | 窗口内显示静态纸条留痕，托盘图标无变化 |
| AC-02 | C2 发出 `character_departed`，角色强度 ≥ 0.3 | 窗口内显示带动画纸条留痕，托盘图标无变化（外出不触发托盘） |
| AC-03 | C2 发出 `character_returned`，角色强度 < 0.3 | 纸条移除，托盘图标无变化 |
| AC-04 | C2 发出 `character_returned`，角色强度 ≥ 0.3 | 纸条移除，托盘图标变色；Fe1 发出 `return_hint_acknowledged` 后恢复正常 |
| AC-05 | 整个外出期间玩家未悬停/点击纸条，角色归来 | `note_unread_on_return` 信号发出 |
| AC-06 | 外出期间多次收到 `leak_content_available` | 窗口内只显示一个角标，不堆叠 |
| AC-07 | Fe2 发出 `leak_content_available`，强度 ≥ 0.7 | 窗口内角标有动画 + 托盘图标变色 |
| AC-08 | 玩家点击泄漏内容区域 | 角标消除，托盘恢复正常 |
| AC-09 | C5 返回强度值 1.5（异常值） | 钳制为 1.0，按 Expressive 档处理，不报错 |
| AC-10 | 窗口隐藏时角色归来，强度 ≥ 0.3 | 托盘图标变色；窗口显示后 Fe1 正常触发 RETURN_HINT |
