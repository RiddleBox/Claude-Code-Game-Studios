# F1 — 桌面窗口系统（Desktop Window System）

> **Status**: Approved
> **Author**: Claude Code Game Studios
> **Last Updated**: 2026-03-26
> **Implements Pillar**: 陪伴不打扰 / 窥视感

## Overview

F1 桌面窗口系统是整个游戏的技术容器。它将 Godot 4 游戏窗口配置为一个透明、无边框、始终置顶、可拖拽的桌面悬浮窗，常驻于玩家桌面角落。所有其他视觉系统（角色动画、UI、演出）都渲染在这个窗口之内。该系统不包含任何游戏逻辑，职责仅限于：窗口的创建、配置、位置管理、尺寸约束、多显示器处理、以及系统托盘集成。

## Player Fantasy

玩家打开电脑开始工作，屏幕角落有一个小小的存在——不是弹窗，不是通知，而是一扇始终在那里的窗口。它不遮挡工作内容，不抢夺焦点，只是静静地存在着。玩家随时可以瞥它一眼，也可以完全无视它继续工作。这种「它在那里，但不打扰我」的感觉，就像桌上放了一个安静陪伴你的小东西——不需要照顾，不会消失，只是在。

## Detailed Design

### Core Rules

#### 窗口配置

1. 窗口默认尺寸：**320×320 px**（这是整个桌宠界面的尺寸，非角色本身大小）
2. 窗口模式：**无边框（borderless）+ 透明背景（transparent）+ 始终置顶（always on top）**
3. 渲染器：强制使用 `gl_compatibility`（兼容透明窗口，避免 D3D12 透明问题）
4. 默认清除颜色：`Color(0, 0, 0, 0)`（完全透明）
5. 窗口不可通过操作系统的任务栏切换聚焦，不出现在 Alt+Tab 列表（`Window.FLAG_NO_FOCUS` 待验证可行性）

#### 窗口位置

6. **首次启动**：窗口默认位置为屏幕**右下角**，距边缘 24px
7. **位置持久化**：窗口位置在退出时写入存档（F4 存档系统），下次启动恢复上次位置
8. 若上次保存的位置超出当前屏幕范围（如更换显示器分辨率），重置到右下角默认位置
9. **拖拽**：玩家可拖拽窗口任意位置（通过 `InputEventMouseMotion` 实现），拖拽结束后立即持久化新位置

#### 窗口尺寸

10. MVP 阶段窗口尺寸**固定为 320×320 px**，不支持缩放
11. 未来版本可开放尺寸调整（预留接口，暂不实现）

#### 系统托盘

12. 游戏启动后在系统托盘显示图标
13. 托盘图标右键菜单：**显示/隐藏窗口**、**设置**、**退出**
14. 托盘图标左键单击：切换窗口显示/隐藏
15. 游戏窗口关闭（点击操作系统关闭按钮）时：**不退出游戏**，改为隐藏窗口，游戏继续在后台运行
16. 真正退出游戏只能通过托盘菜单「退出」或设置界面退出

#### 点击穿透（分阶段实现）

17. **MVP 阶段**：整个 320×320 窗口区域均响应鼠标事件，透明区域也会拦截鼠标点击（不穿透）
18. **Post-MVP**：通过 GDExtension 调用 Win32 API 实现透明像素点击穿透，使鼠标点击透明区域时穿透到下层窗口

#### 多显示器

19. MVP 阶段支持**单显示器**，位置以主显示器坐标系为准
20. 多显示器完整支持延后到垂直切片阶段

### States and Transitions

| 状态 | 进入条件 | 退出条件 | 行为 |
|------|---------|---------|------|
| **Visible** | 游戏启动 / 托盘图标点击显示 | 玩家隐藏窗口 / 托盘切换 | 窗口正常显示，始终置顶，响应拖拽和点击 |
| **Hidden** | 托盘图标切换隐藏 / 点击关闭按钮 | 托盘图标点击显示 | 窗口不可见，游戏逻辑（时间、事件）继续在后台运行 |
| **Dragging** | 玩家在窗口内按下左键并移动 | 玩家释放左键 | 窗口跟随鼠标移动，暂停其他点击事件响应 |
| **Startup** | 游戏进程启动 | 窗口初始化完成 | 读取存档位置，配置窗口属性，切换到 Visible |
| **Shutdown** | 托盘菜单「退出」/ 设置界面退出 | 进程结束 | 持久化当前窗口位置，触发全局存档，退出进程 |

**关键规则**：
- Hidden 状态下游戏逻辑不暂停（Idle 时间继续流逝，事件继续推进）
- Dragging 状态结束时立即写入新位置到存档

### Interactions with Other Systems

| 交互系统 | 数据流向 | F1 提供什么 | F1 需要什么 |
|---------|---------|------------|------------|
| **F4 存档系统** | 双向 | 读取/写入窗口位置 `Vector2i` | 存档读写接口 |
| **C1 角色动画系统** | F1 → C1 | 提供渲染容器（窗口尺寸、根节点） | 无 |
| **P1 主界面 UI** | F1 → P1 | 提供窗口根节点和尺寸常量 | 无 |
| **Fe6 通知/提醒系统** | Fe6 → F1 | 接收「显示/隐藏窗口」指令 | 无 |
| **P3 设置 UI** | 双向 | 接收「退出游戏」指令；提供当前窗口状态 | 无 |
| **F2 角色状态机** | F2 → F1 | 接收 Hidden 状态通知（游戏逻辑不暂停） | 无 |

**接口约定**：
- F1 暴露信号 `window_visibility_changed(is_visible: bool)`，供其他系统感知窗口显隐
- F1 暴露常量 `WINDOW_SIZE: Vector2i = Vector2i(320, 320)`，其他系统引用此常量而非硬编码
- F1 不直接调用任何游戏逻辑系统，只暴露信号和常量

## Formulas

本系统为纯窗口配置系统，无数学公式。

## Edge Cases

**1. 位置超出屏幕边界**
- 条件：上次保存的位置在当前分辨率下超出屏幕（换显示器、改分辨率）
- 处理：重置到右下角默认位置（距边缘 24px）
- 不报错，静默重置

**2. 存档读取失败（首次启动 / 存档损坏）**
- 条件：F4 存档中无窗口位置数据，或数据格式错误
- 处理：使用默认位置（右下角），不阻塞启动流程

**3. 玩家拖拽窗口到屏幕外**
- 条件：拖拽时窗口中心超出屏幕边界
- 处理：不限制拖拽（允许拖到屏幕外，桌宠可以「躲到角落」），位置照常持久化
- 若窗口完全移出屏幕，玩家可通过托盘菜单「重置位置」找回

**4. 游戏进程意外退出（崩溃）**
- 条件：进程被强制终止，未能执行 Shutdown 流程
- 处理：窗口位置在每次拖拽结束时写入（而非只在退出时写入），确保崩溃不丢失位置

**5. 多显示器（MVP 阶段）**
- 条件：玩家有多台显示器
- 处理：MVP 阶段仅支持主显示器，位置以主显示器坐标系为准，不做跨显示器处理

**6. 窗口创建失败（透明不支持）**
- 条件：运行环境不支持透明窗口（极少数 GPU/驱动问题）
- 处理：降级为非透明窗口（有背景色），记录日志，不崩溃

## Dependencies

**上游依赖（F1 依赖的系统）**
- **F4 存档系统**：F1 需要 F4 提供读写接口来持久化窗口位置。F4 尚未设计，接口待定（provisional）。

**下游依赖（依赖 F1 的系统）**
- **C1 角色动画系统**：依赖 F1 提供的渲染容器和 `WINDOW_SIZE` 常量
- **P1 主界面 UI**：依赖 F1 提供的窗口根节点和尺寸常量
- **Fe6 通知/提醒系统**：依赖 F1 的 `window_visibility_changed` 信号和显示/隐藏控制接口
- **P3 设置 UI**：依赖 F1 提供退出游戏接口和当前窗口状态

**外部依赖**
- **Godot 4.6.1** `gl_compatibility` 渲染器：透明窗口的技术基础
- **操作系统系统托盘 API**：通过 Godot `DisplayServer` 或平台插件实现
- **GDExtension（Post-MVP）**：Win32 API 实现透明像素点击穿透

## Tuning Knobs

| 参数 | 当前值 | 安全范围 | 增大效果 | 减小效果 |
|------|--------|---------|---------|----------|
| `WINDOW_SIZE` | `320×320 px` | `240×240` 到 `480×480` | 角色更清晰，占用更多屏幕空间 | 角色更小巧，遮挡更少 |
| `DEFAULT_MARGIN` | `24 px` | `8` 到 `48` | 首次启动距屏幕边缘更远 | 更靠近屏幕边缘 |
| `DRAG_SAVE_DEBOUNCE` | `500 ms` | `100` 到 `2000` | 拖拽结束后更晚写入存档（减少 I/O） | 拖拽结束后更快写入（更安全） |

**注**：MVP 阶段窗口尺寸固定不可调，`WINDOW_SIZE` 为编译期常量。未来版本开放为用户可配置项。

## Visual/Audio Requirements

| 事件 | 视觉反馈 | 音频反馈 | 优先级 |
|------|---------|---------|--------|
| 游戏启动 | 窗口淡入（fade in，约 0.3s） | 无（静默启动，不打扰） | 高 |
| 窗口显示（从 Hidden 恢复） | 窗口淡入（约 0.2s） | 无 | 中 |
| 窗口隐藏 | 窗口淡出（约 0.2s） | 无 | 中 |
| 拖拽中 | 无额外反馈（跟随鼠标即可） | 无 | 低 |
| 降级模式（透明不支持） | 显示半透明深色背景代替透明 | 无 | 低 |

**设计说明**：
- 本系统的视觉反馈极度克制，符合「陪伴不打扰」柱
- 启动淡入是唯一的主动视觉演出，目的是让窗口「出现」而非「弹出」
- 所有声音均为静默——窗口本身不产生任何声音，声音由 Fe5 声音系统管理

**完整版预留需求**：
- 启动时播放模拟屏幕亮起的光效动画（如从黑暗到亮起的光晕），强化「窗口另一侧」的世界观感。具体动画形式待美术方向确定后设计。

## UI Requirements

| 信息 | 显示位置 | 更新频率 | 条件 |
|------|---------|---------|------|
| 系统托盘图标 | 操作系统托盘区域 | 常驻 | 游戏运行时始终显示 |
| 托盘右键菜单：显示/隐藏 | 托盘图标右键菜单 | 按需 | 窗口状态切换时菜单文字更新 |
| 托盘右键菜单：设置 | 托盘图标右键菜单 | 静态 | 始终可用 |
| 托盘右键菜单：退出 | 托盘图标右键菜单 | 静态 | 始终可用 |
| 托盘右键菜单：重置位置 | 托盘图标右键菜单 | 静态 | 始终可用（用于找回移出屏幕的窗口） |

**设计说明**：
- F1 系统本身不在游戏窗口内渲染任何 UI 元素
- 游戏窗口内的所有 UI 由 P1（主界面 UI）和 P3（设置 UI）负责
- F1 只管理操作系统层级的托盘 UI

## Acceptance Criteria

- [ ] 游戏启动后窗口以透明无边框形式显示，无白色/黑色背景
- [ ] 窗口始终置顶，不被其他应用窗口遮挡
- [ ] 窗口可在桌面任意位置拖拽
- [ ] 拖拽结束后关闭并重启游戏，窗口恢复到上次位置（±2px 误差以内）
- [ ] 点击窗口关闭按钮后游戏不退出，窗口隐藏，托盘图标保持显示
- [ ] 托盘图标左键点击切换窗口显示/隐藏
- [ ] 托盘右键菜单包含：显示/隐藏、设置、重置位置、退出
- [ ] 点击「重置位置」后窗口回到右下角（距边缘 24px）
- [ ] 窗口隐藏期间游戏逻辑（时间计数）继续运行，不暂停
- [ ] 分辨率变化后启动，若上次位置超出屏幕则自动重置到右下角
- [ ] 透明窗口不支持时降级为半透明深色背景，游戏不崩溃
- [ ] 窗口启动时有约 0.3s 淡入动画
- [ ] Performance：窗口系统初始化完成时间 < 500ms
