# P3 — 设置 UI（Settings UI）

> **Status**: In Design
> **Author**: Claude Code Game Studios
> **Last Updated**: 2026-03-30
> **Implements Pillar**: 陪伴不打扰（设置轻量、实时生效、不打断体验）

---

## Overview

P3 设置 UI 是玩家调整游戏行为偏好的唯一入口。它以非模态浮层形式覆盖在 P1 之上，提供三类设置：（1）窗口行为设置（位置重置、置顶开关）；（2）声音设置（主音量、音效/环境音分控）；（3）退出游戏（游戏在正常情况下只能通过此处或托盘菜单退出）。第四类 Aria 连接配置（API Key 填写、连接测试、降级模式切换）在 F5 设计完成后补充进 P3。

P3 不持有任何设置数据——它只是设置的视觉入口，写入和读取由 F4 存档系统负责，各系统各自监听配置变化信号。P3 的所有修改实时生效，不需要「保存」按钮。

## Player Fantasy

设置界面不应该让玩家感到「进入了后台管理系统」。它更像是翻开了桌宠窝角落里的一个小本子，里面记着一些可以调整的事——音量大一点、窗口挪个位置、或者连上 Aria 让它能听你说话。

改完就生效，不需要点保存，不需要重启，不需要确认。玩家做完一件事，合上本子，继续工作，一切静悄悄地已经变了。

## Detailed Design

### Core Rules

**开启与关闭**

1. P3 通过 P1 主界面上的设置入口按钮开启，以及托盘菜单「设置」选项开启。
2. P3 以非模态浮层形式覆盖在 P1 之上，玩家可随时点击 P3 外区域关闭。
3. P3 关闭时不做任何额外操作——所有修改已实时生效，无需保存。

**设置项规则**

4. 所有设置修改实时生效，不设「保存」「取消」「应用」按钮。
5. 设置值由 P3 写入 F4 存档（`settings` 键），各系统监听 F4 的 `setting_changed(key, value)` 信号响应变化。
6. P3 开启时从 F4 读取当前设置值初始化各控件状态。

**窗口设置**

7. **重置位置**：点击按钮后窗口立即移动到右下角默认位置（距边缘 24px），调用 F1 接口。
8. **始终置顶开关**：切换后立即生效，写入 F4；默认开启。

**声音设置**

9. **主音量**：滑块控制，范围 0–100，实时调整；写入 F4 `settings.master_volume`。
10. **音效音量**：滑块控制，范围 0–100；写入 F4 `settings.sfx_volume`。
11. **环境音音量**：滑块控制，范围 0–100；写入 F4 `settings.ambient_volume`。
12. 三个音量滑块均显示当前数值，拖动时实时预览效果。

**退出游戏**

13. 点击「退出游戏」按钮后弹出确认对话框（仅此一处使用模态确认），玩家确认后触发 F1 Shutdown 流程。
14. 确认对话框只有「确认退出」和「取消」两个选项，无其他操作。

**Aria 配置（预留）**

15. P3 首版在设置面板底部预留「Aria 连接」区块入口，显示「即将推出」占位文字；F5 设计完成后填充实际配置项。

### Layout

```
┌──────────────────────────┐
│  设置                   ✕ │  ← 标题 + 关闭按钮
├──────────────────────────┤
│  窗口                     │  ← 分组标题
│  [重置窗口位置]             │  ← 按钮
│  始终置顶  [●──────]       │  ← 开关
├──────────────────────────┤
│  声音                     │  ← 分组标题
│  主音量   [────●───] 75   │  ← 滑块 + 数值
│  音效     [──●─────] 50   │
│  环境音   [────────●] 100 │
├──────────────────────────┤
│  Aria 连接  （即将推出）    │  ← 预留区块
├──────────────────────────┤
│        [退出游戏]          │  ← 危险操作，底部独立
└──────────────────────────┘
```

面板固定宽度 260px，高度随内容自适应，不滚动（内容量有限）。

## Edge Cases

| # | 场景 | 处理方式 |
|---|------|----------|
| EC-01 | F4 读取设置失败（首次启动/存档损坏） | 所有控件初始化为默认值（置顶开启、音量均为 75），不报错 |
| EC-02 | 玩家快速拖动音量滑块 | 拖动结束（`drag_ended` 信号）时写入 F4，拖动中只预览音效，不频繁写入 |
| EC-03 | 点击「退出游戏」后确认对话框显示中，角色恰好归来 | 确认对话框保持显示，归来流程在后台正常触发，不被 P3 打断 |
| EC-04 | 托盘菜单「设置」开启 P3 时窗口处于 Hidden 状态 | 先显示窗口（F1 切换到 Visible），再开启 P3 面板 |
| EC-05 | P3 开启期间玩家拖动窗口 | P3 面板跟随窗口移动（作为 P1 子节点，位置相对窗口固定） |

## Dependencies

**上游依赖（P3 依赖这些系统）**

| 系统 | 依赖内容 | 接口 |
|------|---------|------|
| F1 桌面窗口系统 | 重置位置、置顶开关、Shutdown 触发 | 方法 `F1.reset_position()`、`F1.set_always_on_top(bool)`、`F1.shutdown()` |
| F4 存档系统 | 设置值读写 | 方法 `F4.get_setting(key)`、`F4.set_setting(key, value)`；信号 `setting_changed(key, value)` |
| P1 主界面 UI | 提供设置入口按钮与面板锚点 | P1 发出 `settings_opened` 信号触发 P3 显示 |

**下游依赖（这些系统依赖 P3）**

| 系统 | 依赖内容 | 接口 |
|------|---------|------|
| Fe5 声音系统 | 监听 `setting_changed` 响应音量变化 | F4 信号（Fe5 直接监听，不经过 P3） |

## Tuning Knobs

| 参数 | 默认值 | 安全范围 | 影响 |
|------|--------|---------|------|
| `default_master_volume` | 75 | 0–100 | 首次启动默认主音量 |
| `default_sfx_volume` | 75 | 0–100 | 首次启动默认音效音量 |
| `default_ambient_volume` | 75 | 0–100 | 首次启动默认环境音音量 |
| `panel_width` | 260px | 220–300px | 设置面板宽度 |

## Acceptance Criteria

| # | 测试条件 | 通过标准 |
|---|---------|----------|
| AC-01 | 玩家点击 P1 设置入口或托盘「设置」 | P3 面板以浮层形式出现在 P1 之上 |
| AC-02 | P3 开启时 | 所有控件显示 F4 中当前存储的设置值 |
| AC-03 | 拖动主音量滑块到 50，释放 | F4 `settings.master_volume` 写入 50，Fe5 实时响应音量变化 |
| AC-04 | 点击「重置窗口位置」 | 窗口立即移动到右下角（距边缘 24px） |
| AC-05 | 切换「始终置顶」开关 | 窗口置顶状态立即变化，F4 写入新值 |
| AC-06 | 点击「退出游戏」 | 弹出模态确认对话框，包含「确认退出」和「取消」 |
| AC-07 | 确认退出对话框点击「确认退出」 | 触发 F1 Shutdown 流程，游戏退出 |
| AC-08 | 确认退出对话框点击「取消」 | 对话框关闭，P3 面板恢复，游戏继续 |
| AC-09 | 点击 P3 外区域 | P3 整体关闭，设置已生效，无需额外保存 |
| AC-10 | F4 读取失败（模拟损坏存档） | P3 所有控件初始化为默认值，不崩溃 |
