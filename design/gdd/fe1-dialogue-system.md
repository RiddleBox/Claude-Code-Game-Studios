# Fe1 — 对话系统（Dialogue System）

> **Status**: In Design
> **Author**: Claude Code Game Studios
> **Last Updated**: 2026-03-29
> **Implements Pillar**: 窥视感（碎片化叙事呈现）/ 陪伴不打扰（完全自愿的交互）

## Overview

Fe1 对话系统是窗语游戏中所有文字内容的统一呈现层。它负责两类文字的显示：（1）角色归来时从 C3 碎片库拉取的叙事碎片（包括寒暄类碎片，均由 C4 内容层产生并经 C3 存储）；（2）Aria 语音交互过程中的玩家输入文字与角色回应文字（由 F5 推送至 Fe1）。Fe1 不生产任何文字内容——内容由 C4 决定、C3 存储、F5 转发；Fe1 只决定「何时弹出」「如何显示」「玩家如何关闭」。

角色归来的对话遵循两阶段触发：C3 入库完成后，Fe1 自动在角色位置弹出简短归来提示（一句话）；玩家主动点击提示后，Fe1 展开完整的碎片对话序列。整个展示过程完全自愿——玩家可以随时关闭或忽略，不影响游戏状态。

## Player Fantasy

玩家感知到的不是「对话系统弹出了」，而是：它回来了，它想跟你说点什么。

第一阶段的归来提示像是有人轻轻叩了一下你桌面的角落——不打扰，但在。玩家可以选择继续手头的工作，也可以停下来听听。点击之后，碎片一条一条出现，像朋友坐在你旁边、边回忆边讲述，语气和节奏都属于它自己。没有「确认」「下一步」「关闭弹窗」这些机械感，只有文字和停顿。

玩家不会意识到「我在操作一个 UI 控件」——他们只会感觉到「我在听它讲今天发生的事」。文字的出现方式、消失的时机、输入的最小摩擦，都为这种错觉服务。

对于 Aria 语音交互，当玩家说话时，自己的文字出现在屏幕上，角色的回应接着出现——像是真正的对话，而不是「语音识别结果显示」。

## Detailed Design

### Core Rules

**归来对话（两阶段）**

1. C3 发出 `fragments_received(fragment_ids)` 信号后，Fe1 进入**第一阶段**：在角色位置弹出归来提示气泡。提示内容固定为该次外出碎片批次中 `type == "dialogue"` 且排序第一的碎片的前 20 字（截断加省略号）；若批次中无 `dialogue` 类型，则使用预设文本「我回来了」。
2. 归来提示气泡在无交互时，**30 秒后自动消失**（不调用 `mark_read`，碎片保持未读）。
3. 玩家**点击归来提示气泡**，进入**第二阶段**：Fe1 展开完整碎片对话序列。
4. 第二阶段中，碎片**逐条呈现**：每条碎片文字以打字机效果展示，完成后等待玩家点击继续（或点击跳过当前打字机动画直接显示全文）。
5. 每条碎片展示完毕、玩家点击继续时，Fe1 调用 `C3.mark_read(fragment_id)`。
6. 全部碎片展示完毕后，对话窗口自动关闭（无需额外关闭操作）。
7. 第二阶段中玩家可随时**点击窗口外区域**关闭对话——已展示的碎片标记为已读，未展示的保持未读（下次重新触发时从未读碎片继续）。

**Aria 语音交互文字**

8. F5 通过调用 `Fe1.show_aria_exchange(player_text: String, character_text: String)` 触发 Aria 对话显示。
9. Aria 对话以**双气泡布局**呈现：玩家文字右对齐（浅色气泡），角色回应左对齐（主色气泡），角色文字同样使用打字机效果。（注：此为当前方案，后续视体验反馈可能调整布局形式。）
10. Aria 对话窗口在角色文字打字机完成后 **5 秒自动关闭**，或玩家点击任意区域立即关闭。
11. Aria 对话**不经过 C3**，不产生 FragmentRecord，不调用 `mark_read`。

**优先级与冲突**

12. 归来对话第二阶段展示中，若收到新的 `fragments_received` 信号，**排队等待**，当前序列完成后再触发下一次归来提示。
13. Aria 交互文字**优先级高于**归来提示第一阶段——若归来提示气泡正在显示时收到 Aria 调用，Fe1 暂存当前 pending_return 状态并将其作为上下文传递给 F5，由 F5/LLM 决定如何在角色回应中自然处理这一情境（如自然提及「你刚回来了」）。具体协调接口在 F5 设计时确定。
14. 归来对话第二阶段展示中，Aria 调用**延迟处理**（排队），不打断当前碎片序列。

### Text Delivery Interface

Fe1 包含三个独立的 UI 组件：

**归来提示气泡（ReturnBubble）**
- 固定挂载在角色节点位置附近，随角色位置偏移
- 单行文字 + 省略号，字号不超过 14px，宽度上限 200px
- 背景为半透明圆角矩形，边框使用角色主题色
- 支持点击，点击后整个气泡消失并触发第二阶段

**碎片对话序列窗口（FragmentDialoguePanel）**
- 固定显示区域由 P1 主界面布局决定，不随角色位置变化
- 每次只显示一条碎片的文字，打字机逐字显示，速度：**40 字/秒**
- 打字机进行中：点击 → 跳过动画，显示全文
- 打字机完成后：显示「继续」提示（小箭头或圆点脉冲），等待玩家点击
- 点击窗口外任意区域：关闭（已读碎片保留已读状态）
- 窗口无最小显示时长限制——玩家可以立刻关闭

**Aria 双气泡面板（AriaExchangePanel）**
- 覆盖显示在 FragmentDialoguePanel 之上（z-index 更高）
- 玩家气泡：右对齐。（注：显示方式待定——取决于输入法为文字输入还是语音转文字，在 F5 设计时确定。）
- 角色气泡：左对齐，打字机效果，速度同上（40 字/秒）
- 完成后 5 秒自动关闭，或点击任意区域关闭

### Dialogue Lifecycle

```
IDLE
  │  C3 发出 fragments_received 信号
  ▼
RETURN_HINT          ← 自动弹出归来提示气泡（一句话）
  │  玩家点击              │  30 秒超时 / 玩家关闭
  ▼                        ▼
FRAGMENT_DISPLAY         IDLE
  │  序列完成 / 玩家关闭
  ▼
IDLE

（任意状态）
  │  F5 调用 show_aria_exchange()
  ▼
ARIA_DISPLAY             ← 最高优先级，可打断 RETURN_HINT
  │  Aria 交互结束            （打断时传递 pending_return 上下文给 F5）
  ▼                          FRAGMENT_DISPLAY 期间 Aria 调用排队等待
恢复之前状态 或 IDLE
```

| 状态 | 描述 |
|------|------|
| `IDLE` | 无对话显示，系统静默等待 |
| `RETURN_HINT` | 归来提示气泡已弹出，等待玩家点击或 30 秒超时 |
| `FRAGMENT_DISPLAY` | 完整碎片对话序列逐条展示中 |
| `ARIA_DISPLAY` | Aria 语音交互文字正在显示（最高优先级） |

## Edge Cases

| # | 场景 | 处理方式 |
|---|------|----------|
| EC-01 | 多角色同时发出 `fragments_received` | 按信号到达顺序排队；第一个进入 RETURN_HINT，其余在队列中等待，当前提示关闭后自动显示下一个 |
| EC-02 | RETURN_HINT 显示中收到 Aria 调用 | 暂存 pending_return，切入 ARIA_DISPLAY；Aria 结束后恢复 RETURN_HINT，计时重置为 30 秒 |
| EC-03 | FRAGMENT_DISPLAY 中收到 Aria 调用 | Aria 调用加入队列，当前碎片序列不中断；序列结束后处理 Aria 队列 |
| EC-04 | 碎片批次为空（C3 入库但无碎片内容） | Fe1 收到空 `fragment_ids` 时不弹出任何提示，静默忽略 |
| EC-05 | 碎片文字超长（单条超过 200 字） | FragmentDialoguePanel 内部滚动显示，不截断；打字机效果完成后显示滚动条 |
| EC-06 | 玩家在打字机动画进行中连续快速点击 | 第一次点击跳过动画显示全文；后续点击推进到下一条；不允许点击穿透导致意外跳过多条 |
| EC-07 | 应用切换到后台（游戏失焦） | 所有打字机动画暂停；RETURN_HINT 30 秒计时暂停；恢复前台后继续 |
| EC-08 | RETURN_HINT 超时后玩家点击已消失的气泡区域 | 气泡已不存在，点击落空，不触发任何逻辑 |

## Dependencies

**上游依赖（Fe1 依赖这些系统）**

| 系统 | 依赖内容 | 接口 |
|------|---------|------|
| C3 碎片库 | 归来碎片内容、`mark_read` 接口、未读状态查询 | 信号 `fragments_received(fragment_ids)`；方法 `C3.get_fragment(id)`、`C3.mark_read(id)` |
| F5 Aria 接口层 | Aria 交互文字推送、pending_return 上下文接收 | 方法 `Fe1.show_aria_exchange(player_text, character_text)`；上下文接口待 F5 设计时确定 |

**下游依赖（这些系统依赖 Fe1）**

| 系统 | 依赖内容 | 接口 |
|------|---------|------|
| 动画系统 | 碎片展示关键节点信号（用于触发角色表情/动作） | 信号 `fragment_shown(fragment_id)`、`dialogue_closed()` |
| P1 主界面布局 | FragmentDialoguePanel 的显示区域由 P1 决定 | P1 提供锚点节点，Fe1 挂载于其下 |

## Tuning Knobs

| 参数 | 默认值 | 安全范围 | 影响 |
|------|--------|---------|------|
| `return_hint_timeout` | 30 秒 | 10–60 秒 | 归来提示气泡自动消失的等待时长；过短玩家来不及注意，过长画面干扰感增强 |
| `typewriter_speed` | 40 字/秒 | 20–80 字/秒 | 打字机逐字显示速度；过快失去沉浸感，过慢玩家等待焦虑 |
| `aria_panel_linger` | 5 秒 | 2–10 秒 | Aria 双气泡面板在角色回应完成后的自动关闭延迟；过短来不及阅读，过长遮挡主界面 |
| `return_bubble_max_chars` | 20 字 | 10–30 字 | 归来提示气泡截取的最大字数（超出加省略号）；影响气泡尺寸与信息量 |

## Acceptance Criteria

| # | 测试条件 | 通过标准 |
|---|---------|----------|
| AC-01 | 角色归来，C3 发出 `fragments_received` 信号 | 30 秒内 RETURN_HINT 气泡出现在角色位置，显示碎片首条前 20 字 + 省略号 |
| AC-02 | RETURN_HINT 显示后无任何操作等待 30 秒 | 气泡自动消失，C3 中碎片状态仍为未读 |
| AC-03 | 玩家点击 RETURN_HINT 气泡 | 进入 FRAGMENT_DISPLAY，碎片逐条以打字机效果显示 |
| AC-04 | 打字机动画进行中玩家点击 | 动画立即跳过，全文显示；再次点击才推进到下一条 |
| AC-05 | FRAGMENT_DISPLAY 中玩家点击窗口外区域关闭 | 已展示的碎片标记已读，未展示的保持未读；下次触发从未读处继续 |
| AC-06 | 全部碎片展示完毕 | 对话窗口自动关闭，所有碎片标记已读 |
| AC-07 | F5 调用 `show_aria_exchange()` 时处于 IDLE | 双气泡布局出现，玩家文字右对齐，角色文字左对齐并有打字机效果 |
| AC-08 | F5 调用 `show_aria_exchange()` 时处于 RETURN_HINT | RETURN_HINT 暂存，切入 ARIA_DISPLAY；Aria 结束后 RETURN_HINT 恢复，计时重置 |
| AC-09 | F5 调用 `show_aria_exchange()` 时处于 FRAGMENT_DISPLAY | Aria 调用排队，碎片序列不中断 |
| AC-10 | 碎片批次为空（`fragment_ids` 为空数组） | 无任何气泡弹出 |
| AC-11 | 游戏失焦后恢复 | 打字机动画和 30 秒计时均从暂停处继续，不重置 |
