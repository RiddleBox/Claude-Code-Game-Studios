# P2 — 碎片日志 UI（Fragment Log UI）

> **Status**: In Design
> **Author**: Claude Code Game Studios
> **Last Updated**: 2026-03-30
> **Implements Pillar**: 窥视感（碎片化叙事回顾）/ 陪伴不打扰（主动翻阅，不催促）

---

## Overview

P2 碎片日志 UI 是玩家回顾已获得叙事碎片的界面层。它不生成内容，不修改内容，只把 C3 已存储的 FragmentRecord 以可浏览、可重读、可整理的方式呈现给玩家。P2 的首要职责是把原本在 Fe1 中逐条出现、稍纵即逝的归来对话，转化为一个可回看的「日志本」——玩家可以在不被打扰的前提下，自己选择什么时候翻看、重读、联想。

P2 首版仅依赖 C3，提供三个基础能力：（1）按时间倒序浏览全部碎片；（2）按碎片类型筛选；（3）查看单条碎片的完整文本与元数据（获取时间、来源外出批次、是否已读）。P2 不在首版承担世界观扩展、社交流或朋友圈聚合功能；与 Fe3 的整合在后续版本再补充。

P2 依然遵循「陪伴不打扰」原则：它不是强制打开的任务面板，而是玩家主动翻阅的私人记录册。它应该像一本慢慢变厚的小本子，而不是一个充满按钮和红点的系统菜单。

## Player Fantasy

玩家点开日志，不会觉得自己在查数据库，而像是在翻一本被慢慢写满的小册子。

有些句子刚看到时只是奇怪，隔几天再读，却突然明白它原来在暗示什么；有些场景当时没在意，后来因为新的碎片出现，旧内容突然变得重要。日志不是「已收集物列表」，而是一种回想——像朋友以前随口说过的话，被你记在了心里，后来某一天重新想起来，发现意味变了。

这个界面不应该催促玩家完成什么，也不应该给人「我还缺 17 条没收集」的压力。它更像一个安静的抽屉：你想念它、好奇它、或者只是想随手翻翻时，打开它，里面是那些它曾经带回来的只言片语。

## Detailed Design

### Core Rules

**开启与关闭**

1. P2 通过 P1 主界面上的日志入口按钮开启（具体位置由 P1 决定）；P2 本身不决定自己何时出现。
2. P2 以非模态浮层形式覆盖在 P1 之上，不遮挡整个桌面，玩家可随时点击 P2 外区域关闭。
3. P2 关闭时不做任何状态持久化（滚动位置、筛选状态均不保存）。

**碎片列表**

4. P2 从 C3 拉取所有 FragmentRecord，按 `acquired_at` 倒序排列（最新的在最上方）。
5. 列表默认显示全部类型；玩家可按 `type`（dialogue / scene / object / emotion）单选筛选。
6. 每条碎片在列表中显示：碎片类型标签、文本前 40 字预览（超出截断加省略号）、获取时间（相对时间，如「3 天前」）。
7. 未读碎片（`is_read: false`）在列表中有轻微视觉区分（如左侧色块），不使用红点或数字角标。
8. P2 不在列表中显示任何「进度」「总数」「收集率」等数字化指标。

**碎片详情**

9. 玩家点击列表中任意碎片，展开详情视图，显示：完整文本、碎片类型、获取时间（绝对时间）、来源外出批次编号。
10. 详情视图内不提供「上一条/下一条」导航按钮，玩家返回列表后自行选择。
11. 详情视图不调用 `C3.mark_read()`——P2 中的查看不改变已读状态（已读状态只由 Fe1 的展示流程决定）。

**Fe3 预留扩展位**

12. P2 首版不显示任何 Fe3 朋友圈内容；Fe3 整合后，P2 可在列表顶部追加「朋友圈动态」区块，设计时再补充。

### Layout

P2 是一个覆盖在 P1 之上的浮层面板，尺寸约为 280×400px（可在 Tuning Knobs 调整），分为两个视图：

**列表视图**

```
┌─────────────────────────────┐
│  [全部] [对话] [场景] [物件] [情绪]  │  ← 类型筛选标签栏
├─────────────────────────────┤
│ ● dialogue  3天前            │  ← 未读标记（色块）
│ 「他说：等你的人已经不在了……」      │
├─────────────────────────────┤
│   scene     5天前            │
│ 「那条街道的尽头有一盏灯……」         │
├─────────────────────────────┤
│   emotion   7天前            │
│ 「有一刻我突然觉得，那里的时间……」    │
└─────────────────────────────┘
```

**详情视图**（点击任意条目后替换列表区域）

```
┌─────────────────────────────┐
│ ← 返回                        │  ← 返回按钮
├─────────────────────────────┤
│ [dialogue]                   │  ← 类型标签
│                               │
│ 「他说：等你的人已经不在了。    │
│   然后就走了。」               │  ← 完整文本
│                               │
│ 获取于 2026-03-28 14:32       │  ← 绝对时间
│ 来自外出 #outing_007          │  ← 批次编号
└─────────────────────────────┘
```

**布局规则**：
- 面板固定宽度，高度随内容滚动，不超出 P1 窗口边界
- 列表区域支持垂直滚动；详情视图长文本同样支持内部滚动
- 面板位置默认与 P1 窗口左侧对齐弹出（具体方向由实现阶段根据屏幕位置决定）

## Edge Cases

| # | 场景 | 处理方式 |
|---|------|----------|
| EC-01 | C3 无任何碎片（玩家首次开启日志，角色未归来过） | 显示空状态提示文字（如「还没有记录，等它回来吧」），不显示列表 |
| EC-02 | 筛选某类型后该类型无碎片 | 显示该类型空状态提示，不自动切换回「全部」 |
| EC-03 | 碎片文本为空字符串（异常数据） | 预览区显示「—」，详情视图同样显示「—」，不崩溃 |
| EC-04 | P2 开启期间 Fe1 新增碎片（角色恰好归来） | P2 不实时刷新列表；关闭后重新开启才显示新碎片（避免列表跳动打扰阅读） |
| EC-05 | 碎片数量极大（>500 条） | 列表虚拟滚动，只渲染可见区域，不一次性实例化全部节点 |
| EC-06 | P2 开启时窗口靠近屏幕左边缘，左侧无空间 | 面板改为向右侧弹出，自动检测可用空间 |
| EC-07 | 玩家在详情视图时点击 P2 外区域 | P2 整体关闭，不先返回列表再关闭 |

## Dependencies

**上游依赖（P2 依赖这些系统）**

| 系统 | 依赖内容 | 接口 |
|------|---------|------|
| C3 碎片系统 | 所有 FragmentRecord 数据 | 方法 `C3.get_all_fragments() → Array`、`C3.get_fragments_by_type(type) → Array` |
| P1 主界面 UI | 提供日志入口按钮与面板锚点 | P1 发出 `log_opened` 信号触发 P2 显示 |

**下游依赖**：无

## Tuning Knobs

| 参数 | 默认值 | 安全范围 | 影响 |
|------|--------|---------|------|
| `panel_width` | 280px | 240–320px | 日志面板宽度；过窄文字显示困难，过宽遮挡桌面 |
| `panel_height` | 400px | 320–480px | 日志面板高度；受 P1 窗口高度约束 |
| `preview_max_chars` | 40 字 | 20–60 字 | 列表预览截取字数；影响列表密度与可读性 |

## Acceptance Criteria

| # | 测试条件 | 通过标准 |
|---|---------|----------|
| AC-01 | C3 有碎片，玩家开启 P2 | 列表按 `acquired_at` 倒序显示，最新碎片在最上方 |
| AC-02 | C3 无碎片，玩家开启 P2 | 显示空状态提示文字，不崩溃 |
| AC-03 | 玩家选择「对话」筛选标签 | 列表只显示 `type == "dialogue"` 的碎片 |
| AC-04 | 存在未读碎片 | 列表中未读碎片有左侧色块视觉区分，无红点或数字 |
| AC-05 | 玩家点击某条碎片 | 展开详情视图，显示完整文本、类型、绝对时间、批次编号 |
| AC-06 | 在详情视图点击返回 | 回到列表视图，筛选状态保持不变 |
| AC-07 | P2 开启期间角色归来新增碎片 | 新碎片不出现在当前列表；关闭重新开启后出现 |
| AC-08 | 点击 P2 外区域 | P2 整体关闭 |
| AC-09 | P2 中查看碎片详情 | C3 中该碎片的 `is_read` 状态不改变 |
