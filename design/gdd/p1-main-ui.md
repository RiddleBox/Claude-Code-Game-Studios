# P1 — 主界面 UI（Main UI）

> **Status**: Approved
> **Author**: Claude Code Game Studios
> **Last Updated**: 2026-03-29
> **Implements Pillar**: 陪伴不打扰 / 真实存在感

## Overview

P1 主界面 UI 是玩家与窗语唯一的视觉接触面。它是一个分层的合成渲染系统，将四个独立图层（窗口框架、背景、角色、泄漏内容）按 Z 轴顺序叠合，呈现出「屏幕角落有一扇通往异世界的小窗」的核心视觉体验。P1 本身不持有游戏逻辑——它订阅 F2、C2、Fe2 的状态信号，将各系统的状态变化翻译为对应的视觉表现。

P1 管理两种主要视图状态：**常态视图**（角色在窗内，背景正常，可接受玩家互动）和**外出态视图**（角色不在，背景变暗，洞口框架保留，泄漏内容仍可出现）。视图切换由 C2 的出发/归来信号驱动，切换本身是平滑过渡而非瞬间跳变。P1 还管理两种短暂浮层：点击角色触发的**交互菜单**，以及归来时显示的**碎片提示**。

P1 的设计约束核心是「不打扰原则」：所有 UI 元素默认安静存在，不主动抢夺视觉焦点；泄漏内容通过 Fe2 推送的数据包渲染，P1 只负责呈现，不决定内容；玩家的任何交互都是自愿触发，不存在强制弹窗或必须点击的提示。

## Player Fantasy

玩家感知到的不是「UI」，而是一扇窗。它在屏幕角落静静存在，不遮挡工作内容，不弹出提示，不要求注意力。偶尔一瞥，看见里面的小家伙在发呆，或者洞口透出一丝奇怪的光，或者窗口边缘闪过一行看不太清的文字——这一切都轻到像是你自己发现的，而不是游戏在推送给你。

当角色外出时，窗口里面变暗了，但框架还在。玩家知道它还在那里，只是此刻里面没有人。这种「空着但还在」的状态本身就有一种陪伴感——像是你在工作，朋友出门了，但他的房间门还开着。偶尔传来一点模糊的动静，证明那边的世界还在运转。

归来的那一刻有一种小小的惊喜感：背景重新亮起，角色出现，可能带着什么东西回来了。玩家不需要时刻盯着——这种惊喜是在不经意间「发现」的，而不是被通知的。

## Detailed Design

### Core Rules

#### P1 的基本职责

1. P1 是所有游戏视觉内容的唯一渲染出口——角色、背景、泄漏内容、交互菜单、碎片提示均由 P1 负责渲染。
2. P1 不持有游戏逻辑，不决定内容——只订阅信号、接收数据包，将状态翻译为视觉表现。
3. P1 不直接调用其他系统的逻辑方法，只读取暴露的状态数据和订阅信号。

#### 图层规则

4. P1 维护四个固定图层，Z 轴顺序固定，不可动态调整：
   - **Layer 1 窗口框架层**：洞口边框，始终可见，不受任何状态影响
   - **Layer 2 背景层**：异世界环境；Away 时渐暗，归来时渐亮
   - **Layer 3 角色层**：C1 动画渲染区；Away 时由 C1/F2 控制消失
   - **Layer 4 泄漏内容层**：Fe2 推送的文字/特效；始终在最上层

5. 浮层（交互菜单、碎片提示）渲染于 Layer 4 之上，但生命周期独立管理，不影响常驻图层。

#### 交互规则

6. **点击命中区**：角色精灵边界框向外扩展 `HIT_PADDING`（默认 16px）的区域为命中区；点击命中区内触发交互菜单。
7. **拖动区**：点击命中区之外的窗口区域（框架层、背景层）按住拖动可移动整个窗口；拖动不触发交互菜单。
8. **点击穿透**：命中区和拖动区之外的透明像素点击穿透到桌面（由 F1 保证），P1 不处理。

#### 视图切换规则

9. 常态视图 → 外出态视图：收到 C2 `outbound_triggered` 后，背景层在 `AWAY_FADE_DURATION` 内渐暗；角色层由 F2/C1 控制。
10. 外出态视图 → 常态视图：收到 C2 `return_completed` 后，背景层在 `RETURN_FADE_DURATION` 内渐亮；归来碎片提示浮层短暂显示。
11. 所有视图切换为平滑过渡，不做瞬间跳变。

#### 不打扰原则

12. P1 不主动弹出任何全屏遮罩或模态对话框。
13. 交互菜单、碎片提示均为非模态浮层，玩家可无视（点击窗口外自动关闭）。
14. 泄漏内容展示期间，玩家仍可正常拖动窗口和点击其他应用。

### Layer Structure

```
┌─────────────────────────────────────────┐
│  浮层：碎片提示 / 交互菜单               │  Z: 100+  非常驻，生命周期独立
│  Layer 4：泄漏内容叠加层                 │  Z: 30    Fe2 推送，文字/特效
│  Layer 3：角色层                         │  Z: 20    C1 动画节点
│  Layer 2：背景层                         │  Z: 10    异世界环境图，Away 暗场
│  Layer 1：窗口框架层                     │  Z: 1     洞口边框，始终存在
└─────────────────────────────────────────┘
```

#### Layer 1 — 窗口框架层
- **内容**：洞口/窗口边框图形资源
- **行为**：静态，不响应任何状态变化，始终全亮、全不透明
- **尺寸**：与 F1 窗口尺寸一致；F1 `window_resize` 时同步调整
- **交互**：非命中区，点住可拖动窗口

#### Layer 2 — 背景层
- **内容**：异世界环境背景图（MVP 为静态图，后续可扩展为轻微动态）
- **常态**：全亮，`modulate.a = 1.0`
- **Away 过渡**：收到 `outbound_triggered` 后，`modulate` 在 `AWAY_FADE_DURATION`（1.5s）内渐变至 `AWAY_DIM_COLOR`（深色叠加，亮度约 20%）
- **归来过渡**：收到 `return_completed` 后，`modulate` 在 `RETURN_FADE_DURATION`（1.0s）内恢复全亮
- **交互**：非命中区，点住可拖动窗口

#### Layer 3 — 角色层
- **内容**：C1 动画节点的渲染输出
- **P1 职责**：为 C1 节点提供挂载位置和尺寸约束；不直接控制动画
- **Away 时**：C1/F2 控制角色动画切换（Away 状态角色视觉消失），P1 不干预
- **命中区**：角色精灵 `Rect2` + `HIT_PADDING`（16px）扩展，点击触发交互菜单

#### Layer 4 — 泄漏内容叠加层
- **内容**：Fe2 推送的展示数据包，包含：
  - `text`：文字内容 + 持续时间 + 特效参数（`fx_style`）
  - `portal`：洞口光晕特效
  - `visual`：视觉闪现特效
- **P1 职责**：按 Fe2 数据包渲染对应内容；执行淡入/淡出；持续时间结束后自动清除
- **文字渲染**：字体、位置约束（不遮挡命中区）、最大宽度由 P1 管理
- **交互**：透明穿透，不响应点击

#### 浮层 — 交互菜单
- **触发**：玩家点击命中区
- **内容**：MVP 阶段包含 1–3 个简单动作按钮（如「戳一戳」「看看它」）
- **位置**：角色命中区上方，自动避让窗口边界
- **关闭**：点击浮层外任意位置、或 `MENU_TIMEOUT`（5 秒）后自动关闭

#### 浮层 — 碎片提示
- **触发**：C2 `return_completed` 信号
- **内容**：「带回了一些东西」类文字提示 + 可点击展开图标
- **位置**：窗口下方或角色旁，不遮挡角色主体
- **关闭**：点击展开 / `HINT_TIMEOUT`（8 秒）后自动淡出

### State Views

P1 维护两种主要视图状态，由 C2 信号驱动切换。

#### 常态视图（Normal View）

```
┌──────────────────────────┐
│  ░░░░░ 框架边框 ░░░░░     │
│  ┌────────────────────┐  │
│  │  🌄 背景（全亮）    │  │
│  │    [角色动画]       │  │
│  │                    │  │
│  └────────────────────┘  │
└──────────────────────────┘
```

- 背景层：全亮（`modulate = Color(1,1,1,1)`）
- 角色层：C1 正常播放 Idle / Attentive / Reacting 等动画
- 泄漏层：待机（Fe2 可随时推送内容）
- 可交互：点击命中区触发交互菜单，点击其他区域可拖动

#### 外出态视图（Away View）

```
┌──────────────────────────┐
│  ░░░░░ 框架边框 ░░░░░     │
│  ┌────────────────────┐  │
│  │  🌑 背景（暗，20%） │  │
│  │    [角色不在]       │  │
│  │                    │  │
│  └────────────────────┘  │
└──────────────────────────┘
     ↑ 偶尔：Layer 4 泄漏内容出现
```

- 背景层：渐暗至 `AWAY_DIM_COLOR`（亮度约 20%，色调可偏冷/偏蓝，由美术确认）
- 角色层：C1 切换至 Away 状态（角色视觉消失或极弱存在感表现）
- 泄漏层：Fe2 仍可推送泄漏内容，渲染于暗背景之上
- 框架层：完全不变，始终存在
- 交互：命中区消失（无角色可点），窗口整体可拖动

#### 视图切换时序

**常态 → 外出态**（收到 `outbound_triggered`）：
```
t=0.0s  开始背景渐暗 Tween（1.5s）
t=0.0s  C1/F2 同步处理角色 Away 动画（出发动画）
t=1.5s  背景到达 AWAY_DIM_COLOR，外出态完成
```

**外出态 → 常态**（收到 `return_completed`）：
```
t=0.0s  开始背景渐亮 Tween（1.0s）
t=0.0s  C1/F2 同步处理角色 Returning 动画
t=0.5s  碎片提示浮层出现（背景渐亮过半时）
t=1.0s  背景恢复全亮，常态视图完成
```

### Interactions with Other Systems

| 交互系统 | 数据流向 | P1 需要什么 | P1 提供什么 |
|---------|---------|-----------|───────────|
| **F1 桌面窗口系统** | F1 → P1 | `window_resize` 信号（重排布局）；`window_hidden` / `window_shown` 信号 | — |
| **F2 角色状态机** | F2 → P1 | `state_changed` 信号（Away 时禁用命中区） | — |
| **C1 角色动画系统** | C1 ↔ P1 | C1 节点挂载于 Layer 3；`get_character_bounds() -> Rect2`（计算命中区） | Layer 3 挂载点位置和尺寸约束 |
| **C2 外出-归来循环** | C2 → P1 | `outbound_triggered()` 信号；`return_completed()` 信号 | — |
| **Fe2 泄漏内容系统** | Fe2 → P1 | 展示数据包（content, duration, fx_style, display_type） | Layer 4 渲染区域位置和尺寸 |
| **Fe5 声音系统** | P1 → Fe5 | — | 交互菜单点击时触发 UI 交互音（Post-MVP） |
| **Fe6 通知系统** | Fe6 → P1 | 通知展示请求（Post-MVP） | — |
| **F4 存档系统** | 双向 | 读取窗口上次位置 | 窗口位置变更时写入存档 |

**接口约定：**
- P1 暴露方法 `show_leak_content(packet: LeakDisplayPacket) -> void`，供 Fe2 调用推送泄漏内容
- P1 暴露方法 `show_return_hint(fragment_count: int) -> void`，供 C2 调用显示碎片提示
- P1 不暴露图层直接访问接口，外部系统不直接操作 P1 的子节点

## Formulas

P1 为纯表现系统，无游戏逻辑公式。以下为布局计算和过渡动画的参数规则：

#### 命中区计算

```
hit_rect = character_bounds.grow(HIT_PADDING)

变量定义：
  character_bounds   C1.get_character_bounds() 返回的角色精灵 Rect2
  HIT_PADDING        命中区向外扩展像素数，默认 16px

示例（假设角色精灵 Rect2(40, 60, 120, 160)）：
  hit_rect = Rect2(24, 44, 152, 192)
```

#### 背景暗场颜色计算

```
away_modulate = Color(r, g, b, 1.0)
  其中 r, g, b 由 AWAY_DIM_COLOR 定义

AWAY_DIM_COLOR 默认值：Color(0.15, 0.18, 0.25, 1.0)
  效果：亮度约 20%，轻微偏冷蓝色调，营造「里面熄灭了」的感觉
  （具体色值由美术方向确认后调整）

Tween 插值：
  常态 → 外出态：Color(1,1,1,1) → AWAY_DIM_COLOR，时长 AWAY_FADE_DURATION（1.5s），缓动 ease_in_out
  外出态 → 常态：AWAY_DIM_COLOR → Color(1,1,1,1)，时长 RETURN_FADE_DURATION（1.0s），缓动 ease_out
```

#### 文字泄漏内容位置计算

```
text_position.x = clamp(random_x, TEXT_MARGIN, window_width - TEXT_MARGIN - text_width)
text_position.y = clamp(random_y, TEXT_MARGIN, window_height - TEXT_MARGIN - text_height)

约束：文字区域不与 hit_rect 重叠（若重叠则向上偏移至 hit_rect.top - TEXT_MARGIN）

变量定义：
  TEXT_MARGIN   文字距窗口边缘最小距离，默认 12px
  random_x/y   Fe2 数据包中的位置建议值（若无则随机）
```

## Edge Cases

**1. C1 节点未就绪时收到 Fe2 泄漏数据包**
- 条件：C1 加载失败，`get_character_bounds()` 返回空 Rect2
- 处理：命中区设为空（禁用交互），泄漏文字位置降级为窗口中央，继续渲染，记录警告日志

**2. 背景 Tween 进行中收到反向信号**
- 条件：正在渐暗过程中收到 `return_completed`（极端情况：出发后立即归来）
- 处理：立即取消当前 Tween，以当前 modulate 为起点启动渐亮 Tween，不跳变

**3. 泄漏文字与命中区重叠**
- 条件：Fe2 推送的文字位置恰好覆盖角色命中区
- 处理：按 Formulas 中的约束自动上移文字位置；若窗口高度不够则改为下移

**4. 窗口隐藏时收到视图切换信号**
- 条件：`window_hidden` 期间收到 `outbound_triggered` 或 `return_completed`
- 处理：记录状态变更，暂停 Tween；`window_shown` 后以目标状态直接呈现（不补播过渡动画）

**5. 碎片提示与泄漏内容同时显示**
- 条件：`return_completed` 触发碎片提示时，Layer 4 仍有泄漏内容在播放
- 处理：碎片提示作为浮层独立显示，不干扰 Layer 4；两者可同时存在，碎片提示位置自动避让泄漏内容区域

**6. 交互菜单显示期间收到 `outbound_triggered`**
- 条件：玩家正在查看交互菜单，角色突然出发
- 处理：立即关闭交互菜单，禁用命中区，开始背景渐暗过渡

**7. 窗口尺寸极小时 UI 元素溢出**
- 条件：F1 允许的最小窗口尺寸下，文字/浮层超出窗口边界
- 处理：所有渲染元素强制 clamp 在窗口 Rect2 内；文字若超出则截断并加省略号

**8. F4 存档无窗口位置记录（首次启动）**
- 条件：新玩家首次启动，无历史位置数据
- 处理：窗口初始位置设为屏幕右下角（距边缘 `DEFAULT_MARGIN`，默认 24px）

## Dependencies

| 依赖系统 | 依赖方向 | P1 需要什么 | P1 提供什么 |
|---------|---------|-----------|───────────|
| **F1 桌面窗口系统** | F1 → P1 | `window_resize` / `window_hidden` / `window_shown` 信号；透明窗口和点击穿透由 F1 保证 | — |
| **F2 角色状态机** | F2 → P1 | `state_changed` 信号（P1 依据 Away 状态禁用/启用命中区） | — |
| **C1 角色动画系统** | C1 ↔ P1 | `get_character_bounds() -> Rect2`；C1 节点挂载于 P1 Layer 3 | Layer 3 挂载点；窗口尺寸约束 |
| **C2 外出-归来循环** | C2 → P1 | `outbound_triggered()` / `return_completed()` 信号 | `show_return_hint(fragment_count)` 方法 |
| **Fe2 泄漏内容系统** | Fe2 → P1 | `show_leak_content(packet)` 方法调用 | Layer 4 可用渲染区域（Rect2） |
| **F4 存档系统** | 双向 | 启动时读取窗口位置 | 窗口拖动结束时写入新位置 |

**反向依赖（依赖 P1 的系统）：**

| 系统 | 期望 P1 提供什么 |
|------|----------------|
| Fe2 | `show_leak_content()` 方法可调用；Layer 4 渲染区域不为空 |
| C2 | `show_return_hint()` 方法可调用 |
| C1 | Layer 3 挂载点存在，尺寸约束合理 |
| Fe6（Post-MVP） | 通知渲染区域可用 |

## Tuning Knobs

| 参数 | 默认值 | 安全范围 | 影响 |
|------|--------|---------|------|
| `HIT_PADDING` | 16px | 8–32px | 命中区大小；过小难以点击，过大误触率高 |
| `AWAY_FADE_DURATION` | 1.5s | 0.5–3.0s | 外出背景渐暗速度；过快显得突兀，过慢玩家不确定是否出发 |
| `RETURN_FADE_DURATION` | 1.0s | 0.3–2.0s | 归来背景渐亮速度；应比渐暗快，强调「回来了」的惊喜感 |
| `AWAY_DIM_COLOR` | Color(0.15, 0.18, 0.25, 1.0) | 亮度 10–35% | 外出态背景暗度；过暗失去窗口存在感，过亮不像「里面没人」 |
| `MENU_TIMEOUT` | 5s | 3–10s | 交互菜单自动关闭等待时间；过短玩家来不及选择，过长遮挡视野 |
| `HINT_TIMEOUT` | 8s | 5–15s | 碎片提示自动淡出时间；过短玩家可能错过，过长持续占据视野 |
| `TEXT_MARGIN` | 12px | 6–20px | 泄漏文字距窗口边缘最小距离；过小文字贴边难读 |
| `DEFAULT_MARGIN` | 24px | 12–48px | 首次启动窗口距屏幕边缘距离 |
| 碎片提示出现时机 | 归来 Tween 过半（0.5s） | 0.0–1.0s | 提示出现与背景渐亮的相对时机；过早显得仓促，过晚与归来感脱节 |

## Acceptance Criteria

**图层渲染**
- [ ] 启动时四个图层按 Z 轴顺序正确叠合：框架(1) → 背景(2) → 角色(3) → 泄漏内容(4)
- [ ] 窗口框架层在任何状态下始终全亮、全不透明

## Open Questions

**1. 背景层视觉风格（美术方向待定）**

`AWAY_DIM_COLOR` 的具体色值需美术方向确认：冷蓝色调是默认建议，但实际角色的世界观（奇幻/机械/自然）会影响暗场色调选择。此外，背景图是静态图还是轻微动态（如粒子、光影晃动）留待 P1 实现阶段由美术方向决定。
- 负责人：美术方向
- 目标层级：P1 垂直切片阶段确认

**2. 交互菜单的具体动作内容**

MVP 阶段交互菜单包含哪些按钮（如「戳一戳」「看看它」「摸摸头」）尚未确定，需要与游戏设计和叙事方向协调，确保动作内容与角色性格和世界观一致。
- 负责人：游戏设计 + 叙事方向
- 目标层级：P1 垂直切片阶段，与 C5 性格系统协调

**3. 碎片提示的展开行为**

玩家点击碎片提示后的展开行为归属尚未确定：是 P1 内联展开（简单文字），还是跳转到 Fe1 对话系统，还是 P2 碎片日志 UI？MVP 阶段建议 P1 内联展开，后续系统就绪后迁移。
- 负责人：开发者
- 阻塞：Fe1 / P2 设计完成后确认最终归属

**4. 多显示器和高 DPI 屏幕适配**

窗口位置存档使用像素坐标，在不同分辨率/DPI 的显示器间迁移时可能出现窗口位置异常。需确认 F1 是否处理 DPI 缩放，P1 的坐标系是否需要归一化。
- 负责人：开发者（F1 + P1 集成测试时验证）

**5. 窗口框架和背景图的美术资源规格**

Layer 1 和 Layer 2 的图形资源尺寸、格式、透明通道要求尚未与美术方向对齐。需在 P1 实现前确定资源规格（尺寸是否固定、是否支持拉伸、透明区域的处理方式）。
- 负责人：美术方向 + 开发者
- 目标层级：P1 实现前
