# Game Concept: 窗语（Window Whisper）

*Created: 2026-03-26*
*Status: Draft*

---

## Elevator Pitch

> 一款常驻桌面的陪伴型 Idle 游戏。你的屏幕角落有一扇通往异世界的小窗口，窗口另一侧住着一个非人类的小家伙——它有自己的朋友圈、自己的故事、自己的生活。它会出门探险、拜访朋友、经历各种事，归来时带给你另一个世界的碎片。时间久了，它会慢慢变得更像你身上最好的那部分。

## Core Fantasy

你没有在「玩」一个游戏——你打开电脑工作，然后意外发现屏幕角落有一扇小窗口。窗口另一端有个小东西在看着你，它来自你不了解的某个地方，有着你不理解的生活。但它选择留在这扇窗口旁边陪着你。

你是它世界的神秘观察者，它是你世界的温柔闯入者。两个世界之间只有这一扇小窗，却足以让两个存在彼此改变。

---

## Unique Hook

**像桌宠游戏，但你不是在养它——你们在彼此陪伴。**

- 它有自己的生活，会「出门」，你只能从泄漏进来的声音碎片猜测它遭遇了什么
- 它的成长不是数值提升，而是性格层面的内在变化——被你影响，吸收你的特质
- AI 语音辅助无缝嵌入：出门时窗口随身携带，辅助能力全程不中断
- 故事由事件线 × 性格变量共同决定，不同性格的角色面对同样的世界会走出完全不同的故事

---

## Core Verb

**窥视（Peek）** — 玩家最常做的事是「瞥一眼屏幕角落，看看它在做什么，发生了什么」。

不是操作，不是管理，是带着好奇心的凝视。

## Core Loop

### 30 秒循环（时刻感知）

```
角色常驻角落（存在感）
  → 偶尔有小动画/声音/文字泄漏（勾起好奇）
  → 玩家瞥一眼（主动或被动感知）
  → 继续工作（不打断节奏）
```

### 5 分钟循环（交互时刻）

```
角色归来 / 事件触发
  → 带回故事碎片（对话、物件、只言片语）
  → 玩家主动点击展开或忽略（完全自愿）
  → 碎片积累 → 关系深化 / 世界拼图补全
```

### 长线循环（叙事弧线）

```
足够多的碎片 + 足够多的相处时光
  → 解锁事件线节点（角色个人成长事件、环境变动、世界大事件）
  → 角色依据性格特质做出独特反应
  → 性格在相处中受玩家影响，产生内在变化（共鸣成长）
  → DLC / 更新带来新的事件线包
```

---

## 外出系统（Out & Return）

角色定期「外出」——拜访朋友圈成员、处理自己世界的事务、经历随机事件。

**外出期间**：
- 角色不在画面中心，但窗口随身携带（世界观待定：魔法物件 / 机械终端 / 天生能力）
- AI 语音辅助功能全程不中断（对玩家工作的辅助继续运行）
- 泄漏内容以**纯文字/声效**形式偶尔出现：打斗声、只言片语、惊呼、笑声——唤起好奇，不直接揭示

**归来时**：
- 触发碎片演出（对话片段、带回的小物件、一句感慨）
- 不强制玩家关注，可以点开也可以忽略，稍后在日志中查看

## 角色设计

### 外形
- 非人类形象，外形讨喜
- 通过「召唤/邂逅」机制随机生成（具体机制待定）
- 外形风格方向：机械、奇幻生物、或其他异世界美学（待定，可随机）

### 性格
- 通过随机系统生成性格特质（开朗/怯懦/好奇/慵懒/热血……）
- 本质上都是善良的——即使「邪恶」也是蠢萌的邪恶
- 性格是叙事分支的核心变量：同一事件，不同性格的反应截然不同

### 成长（共鸣成长）
- 不是数值成长，是性格层面的内在变化
- 通过足够多的共同经历和对话，逐渐吸收玩家的正向特质
- 成长表现：对话语气变化、处理事件的方式变化、与朋友圈互动方式变化

### 朋友圈
- 角色有自己的朋友圈（2-5 个配角）
- 每个朋友有独立性格和背景，有彼此之间的过去与正在发生的故事
- 朋友圈是世界观传递的载体，也是事件线的触发来源

## 故事结构

### 事件线设计
- **精心策划的事件节点**：角色个人成长节点、环境变动（搬家、季节、世界事件）、朋友圈的重要事件
- **性格变量**：同一事件线，角色性格不同 → 反应不同 → 故事走向不同
- **日常延伸**：事件线触发后，相关日常碎片持续产生，形成余波

### 内容扩展策略
- 基础版：完整的初始事件线（角色相遇 → 建立羁绊 → 第一次重大事件 → 共鸣成长节点）
- DLC / 更新：新的事件线包（新的生活阶段、新的朋友圈故事、世界大事件）
- 远期：创意工坊支持（暂不纳入当前范围）

## MDA 分析

| 层次 | 内容 |
| ---- | ---- |
| **Mechanics** | 出门-归来循环、碎片收集、事件线触发、性格变量系统、共鸣成长计数 |
| **Dynamics** | 好奇心驱动的窥视行为、碎片拼图的满足感、与角色的情感依附、对「它现在在干什么」的持续想象 |
| **Aesthetics** | **Fellowship**（陪伴感）为主，**Discovery**（发现异世界碎片）为辅，**Narrative**（故事感）贯穿全程 |

## 设计柱

| # | 柱名 | 定义 | 设计测试 |
|---|------|------|----------|
| 1 | **窥视感** | 玩家永远只能看到异世界的一个小角落，信息是碎片化、暗示性的，而非全景展示 | 若某功能直接展示异世界全貌 → 违反此柱 |
| 2 | **真实存在感** | 角色有自己的生活、朋友圈和情绪，它的世界在玩家看不到的时候也在运转 | 若角色只在玩家互动时才「活着」 → 违反此柱 |
| 3 | **性格即命运** | 角色的随机性格特质决定它对相同事件的不同反应，性格是叙事分支的核心变量 | 若两个不同性格的角色对同一事件反应相同 → 违反此柱 |
| 4 | **陪伴不打扰** | 游戏常驻桌面但不强制注意力，玩家可以专注工作，有趣的事才会轻柔地吸引你 | 若某功能需要玩家停下工作才能触发 → 违反此柱 |
| 5 | **治愈日常** | 即使是「坏事」也以蠢萌的方式呈现，整体基调永远是温暖和善意的 | 若某事件让玩家感到焦虑或压力 → 违反此柱 |
| 6 | **共鸣成长** | 角色的成长不是数值强化，而是通过共同经历逐渐吸收玩家的正向特质，产生性格层面的内在变化 | 若某「成长」表现为数值提升或战斗强化 → 违反此柱 |

### 反柱（我们不做的事）

- **不做收集竞争**：不是集卡游戏，角色不是可量化比较的资产
- **不做实时操作**：不需要玩家停下工作来操作战斗或解谜
- **不做负罪感机制**：不存在「放置太久角色会生气/死亡/离开」的惩罚
- **不做全景世界展示**：永远不直接展示异世界的宏观面貌
- **不做数值成长**：角色不变强，只变得更有深度

## 玩家类型分析

### 主要目标玩家
**Explorer（探索者）+ Storyteller（故事爱好者）**
- 喜欢发现碎片、拼凑世界观的乐趣
- 享受叙事积累带来的情感满足
- 不需要竞争或挑战，享受发现本身

### 次要受众
**Fellowship 型玩家** — 渴望陪伴感、喜欢虚拟关系建立的玩家

### 不适合的玩家
- 需要实时挑战和操作反馈的玩家
- 追求数值成长和战斗爽感的玩家
- 希望快速获得大量内容的玩家

### 市场参照
- Neko Atsume（猫咪后院）：碎片发现 + 无操作陪伴，验证了该受众的存在
- Chill with you: Lo-Fi Story：视觉陪伴感验证，但窗口化体验复杂度过高
- Virtual Youtuber / AI 伴侣类产品：验证了玩家对「有性格的AI陪伴」的需求

## 技术集成：Aria 接口

本项目负责范围：
- 交互界面与视觉演出
- 角色性格、动画、情绪表达系统
- 故事碎片与事件线系统
- 外出-归来循环与泄漏内容系统

**Aria 项目负责范围**（外部依赖）：
- 语音识别与自然语言处理
- AI 工作辅助功能
- 跨项目通信接口

**接口边界**：本项目通过定义好的 API 接口与 Aria 通信，不直接实现 AI 功能。接口设计需确保：Aria 不可用时游戏核心陪伴功能仍可正常运行（降级模式）。

## 最大风险

| 风险类型 | 描述 | 缓解策略 |
|---------|------|----------|
| **技术风险** | Godot 4 桌面悬浮窗实现（透明窗口、始终置顶、跨应用层级）可能有平台限制 | 早期技术原型验证，优先级最高 |
| **设计风险** | 「不打扰」与「足够吸引注意」之间的平衡极难拿捏 | 早期用户测试，设立明确的注意力打断阈值指标 |
| **叙事风险** | 性格变量 × 事件线的组合爆炸导致内容量超出个人开发能力 | MVP 阶段限制性格特质数量（3-4种），事件线数量（1条主线） |
| **集成风险** | Aria 接口变更影响本项目功能 | 明确接口契约，实现完整降级模式 |
| **市场风险** | 桌宠类游戏受众小众，定价能否支撑开发成本 | 早期 Steam 心愿单测试市场热度 |

## MVP 定义

**MVP 假设**：玩家会对「有自己生活的异世界桌宠」产生情感依附，并愿意为这种陪伴体验付费。

**MVP 必须验证**：核心陪伴感 + 出门-归来循环是否足够吸引人，哪怕没有深度故事内容。

**MVP 必须包含**：
1. 透明悬浮窗技术实现（Godot 4 桌面窗口）
2. 角色基础动画与存在感表现（待机、互动、出门状态）
3. 出门-归来循环（含泄漏内容机制）
4. 最少1条事件线（约5-8个节点）
5. 基础对话/碎片系统
6. Aria 接口的最小降级模式

**MVP 明确不包含**：
- 随机角色生成系统（使用固定角色）
- 共鸣成长系统（仅设计框架，不实现）
- 朋友圈（仅通过对话提及）
- DLC 系统

### 范围层级

| 层级 | 内容 | 功能 | 目标 |
| ---- | ---- | ---- | ---- |
| **MVP** | 1个固定角色，1条事件线 | 悬浮窗+出门循环+基础对话 | 验证核心陪伴感 |
| **垂直切片** | 1个角色，完整初始事件线 | 完整碎片系统+朋友圈引入 | 验证叙事深度 |
| **Alpha** | 随机角色系统，多条事件线框架 | 性格变量系统+共鸣成长 | 验证重玩价值 |
| **完整版** | 完整内容+Aria集成+润色 | 所有核心功能+降级模式 | 可发行状态 |
