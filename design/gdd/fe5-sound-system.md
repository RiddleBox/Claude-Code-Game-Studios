# Fe5 — 声音系统（Sound System）

> **Status**: Approved
> **Author**: Claude Code Game Studios
> **Last Updated**: 2026-03-29
> **Implements Pillar**: 真实存在感（声音让角色活着）/ 陪伴不打扰（声音克制，绝不突然）

## Overview

Fe5 声音系统负责窗语中所有角色音效的播放与音量管理。它订阅来自 F2、C1、C2、Fe2 的信号，将游戏事件映射为对应的音效播放指令，并统一管理全局音量和静音开关——这是「陪伴不打扰」原则的基础设施：玩家随时可以一键静音，声音永远不会突然变响。Fe5 是纯音效层，不处理背景音乐（BGM 留待后续系统扩展）。

Fe5 管理四类音效：**状态音效**（F2 状态切换时的情绪/环境音）、**动画帧音效**（C1 关键帧触发的逐帧音效，如脚步、翅膀）、**泄漏音效**（Fe2 泄漏内容的混响处理音）、**事件音效**（C2 出发/归来等游戏事件音）。所有音效均受全局音量和静音开关控制，且默认音量保守——宁可偏小，不可突然刺耳。

## Player Fantasy

玩家不会把 Fe5 感知为「音效系统」，而会感知为：屏幕角落那个小家伙真的在这里。它偶尔有轻微的动静，探出时带一点空气感，归来时有轻轻的落点声，外出时偶尔传来模糊又遥远的异世界声响。声音不应该像游戏提示音那样跳出来打断工作，而应该像桌边某个真实存在发出的细小动静——你不一定每次都注意到，但一旦听见，就会更相信它活着。

当角色外出时，声音还承担另一层作用：让「另一个世界正在发生什么」被隐约感知到。不是完整叙述，而是模糊的金属碰撞、远处笑声、被混响和低通包住的一小段回应。玩家听到的不是信息本身，而是一种「刚刚错过了什么」的感觉。Fe5 的目标不是吸引玩家必须立刻看，而是让玩家在不被打扰的前提下，偶尔因为一声轻响而产生一点点好奇和陪伴感。

## Detailed Design

### Core Rules

#### Fe5 的基本职责

1. Fe5 是窗语中所有**角色相关音效**的唯一管理者，负责：
   - 接收各系统的音效触发信号
   - 将事件映射为对应音效资源
   - 控制播放、停止、替换和淡出
   - 管理全局音量与静音状态

2. Fe5 **不负责**背景音乐（BGM）、系统通知音或操作系统层声音；这些不在 MVP 范围内。

3. Fe5 只处理**音效表现层**，不改变任何游戏状态，不参与逻辑判定。

#### 音效分类并发规则

4. Fe5 将音效分为四类：状态音效、动画帧音效、泄漏音效、事件音效。

5. 不同类别可并发播放，每类有并发上限：状态音效 1 条，动画帧音效 2 条，泄漏音效 1 条，事件音效 1 条。

6. 超限替换规则：状态音淡入替换旧音；动画帧音忽略最短的保留最新；泄漏音直接替换；事件音按优先级覆盖。

#### 音量与静音规则

7. 所有音效播放前先乘以 `master_sfx_volume`（0.0–1.0），默认值保守偏低。

8. 静音时不启动新播放，已播音效 0.1 秒内淡出。

9. 默认策略**宁小勿大**：所有默认音量以「工作场景下不打扰」为基准。

#### 播放映射规则

10. 外部系统只能发事件信号，由 Fe5 内部映射资源，不接受通用播放接口。

11. 查表找不到映射时静默跳过，记录调试日志，不崩溃。

12. `play_leak_audio` 的 `effect` 只能使用 Fe5 支持的有限预设（如 `reverb_low`）。

13. 外出期间泄漏音效默认加**混响 + 低通滤波**，营造「隔着空间传来」的感觉。

### Sound Categories

Fe5 将所有音效按功能和感知目标划分为四类，每类有独立的资源池、音量基准和并发策略。

#### 1. 状态音效（State SFX）
- **来源**：F2 `state_changed(old, new)`
- **用途**：标记角色状态切换，让玩家感受到它「活着」、在变化
- **表现原则**：以短音效为主（通常 < 1 秒）；音色偏轻、软、不过分强调 UI 感；不做夸张提示音
- **示例**：Idle 轻微整理动静；Attentive 注意到玩家的轻响；Talking 极轻的开口气息前奏

#### 2. 动画帧音效（Animation SFX）
- **来源**：C1 `animation_event(event_name)`
- **用途**：增强逐帧动作的重量感、材质感和生命感
- **表现原则**：与关键帧严格同步；单次很短；允许同一动作触发 1-2 个分层音
- **示例**：探出动作落定；翅膀/尾巴摆动；Returning 落地碰撞点

#### 3. 泄漏音效（Leak SFX）
- **来源**：Fe2 `play_leak_audio(audio_id, effect)`
- **用途**：让玩家隐约感知角色在异世界的遭遇
- **表现原则**：默认带混响 + 低通滤波；音量明显低于普通事件音；强调朦胧感；不单独承担明确叙事
- **示例**：模糊金属碰撞；远处笑声/惊呼；环境异响、风声

#### 4. 事件音效（Event SFX）
- **来源**：C2 出发/归来等关键事件
- **用途**：给外出循环关键节点一个可感知的边界
- **表现原则**：比状态音效更明确但仍克制；可使用更完整的音头+尾音结构（1–2 秒）
- **示例**：出发轻柔离开声（风声渐远）；归来轻落点、亮起、碰撞轻响

#### 5. 持续底音（Ambient Loop，受限）
- Fe5 原则上不为大多数状态播放持续 loop 音
- MVP 阶段仅允许 Away 状态存在极轻微的持续底音候选（如微风、洞口空鸣）
- 必须满足：音量极低；可被静音立刻关闭；不影响玩家长时间工作
- 若测试发现造成疲劳或注意力打断，则 MVP 直接禁用

### States and Transitions

| F2 状态 | 进入时 Fe5 行为 | 退出时 Fe5 行为 | 持续底音 |
|---------|---------------|---------------|----------|
| Idle | 播放 `sfx_state_idle`（低频触发，不是每次都播） | — | 无 |
| Attentive | 播放 `sfx_state_attentive`（< 0.5s 轻响） | — | 无 |
| Interacting | 播放 `sfx_state_interact` | — | 无 |
| Talking | 播放 `sfx_state_talking_start`（极轻气息前奏） | — | 无 |
| Reacting | 播放 `sfx_state_react`（与反应类型对应） | — | 无 |
| Performing | 播放 `sfx_state_perform_start` | — | 无 |
| Away | 播放 `sfx_event_departure`；启动底音（若配置） | 底音 0.3s 淡出 | 可选极轻底音 |
| Returning | 播放 `sfx_event_return`；停止底音 | — | 无 |

**补充规则：**
- Idle 的状态音**不是每次进入都播**，而是在 F2 自然回落到 Idle 且距上次播放超过 `IDLE_SFX_COOLDOWN`（默认 60 秒）才播，避免频繁触发
- 状态切换时旧状态音效淡出时长 0.15 秒
- Away → Returning 时底音与归来事件音可短暂重叠，底音 0.3s 淡出

## Formulas

Fe5 为纯表现系统，无游戏逻辑公式。以下为实际播放音量的计算规则：

#### 实际播放音量计算

```
final_volume = asset_base_volume * master_sfx_volume * category_modifier

变量定义：
  asset_base_volume     音效资源自身基准音量，范围 0.0–1.0，由音频资源标定
  master_sfx_volume     玩家设置的全局音效音量，范围 0.0–1.0，默认 0.5
  category_modifier     各类别音量修正系数（见下表）

category_modifier 默认值：
  状态音效：   1.0
  动画帧音效： 0.8
  泄漏音效：   0.6   ← 刻意压低，强调朦胧远距感
  事件音效：   1.0
  持续底音：   0.3   ← 极轻，长时间存在不造成疲劳

示例：
  泄漏音效 asset_base_volume=0.8，master_sfx_volume=0.5，category_modifier=0.6
  final_volume = 0.8 × 0.5 × 0.6 = 0.24

  归来事件音 asset_base_volume=0.7，master_sfx_volume=0.5，category_modifier=1.0
  final_volume = 0.7 × 0.5 × 1.0 = 0.35
```

#### reverb_low 效果参数

```
reverb_low 预设定义：
  混响湿度（wet）：  0.4   范围 0.0–1.0
  低通截止频率：     800Hz 范围 200–4000Hz
  混响衰减时间：     1.2s  范围 0.5–3.0s

效果目标：让声音听起来像「隔着一道空间传来」，不是完全失真，但明显不在同一个房间。
```

## Edge Cases

**1. 查表找不到对应音效映射**
- 条件：`state_changed` 或 `animation_event` 的值在映射表中无对应条目
- 处理：静默跳过，记录调试日志，不崩溃，不影响其他音效播放

**2. 音效资源文件缺失**
- 条件：映射到的音频文件未加载或路径无效
- 处理：静默跳过，记录警告日志，不崩溃

**3. 窗口隐藏时收到音效触发**
- 条件：F1 发出 `window_hidden` 后仍收到音效信号
- 处理：静默丢弃所有新触发；已播音效立即淡出（0.1 秒）

**4. 静音状态下收到音效触发**
- 条件：`set_mute(true)` 已调用
- 处理：不播放，不排队，直接丢弃；不记录日志（正常行为）

**5. Away 底音与归来事件音同时触发**
- 条件：`return_completed` 到来时底音仍在播放
- 处理：底音 0.3 秒淡出，归来事件音立即播放；短暂重叠可接受，不视为冲突

**6. `play_leak_audio` 携带不支持的 effect 值**
- 条件：Fe2 传入 Fe5 未定义的 effect 预设名
- 处理：降级为无特效直接播放，记录警告日志

**7. 存档读取失败时音量初始化**
- 条件：F4 存档损坏或首次启动无存档
- 处理：`master_sfx_volume` 使用默认值 0.5，静音状态默认关闭，继续运行

**8. 快速连续状态切换（< 0.15 秒内多次 state_changed）**
- 条件：F2 在极短时间内连续切换状态（如 Idle → Attentive → Idle）
- 处理：每次切换均触发淡出+新音，若淡出未完成则直接以当前音量为起点开始新淡出；不累积队列

## Tuning Knobs

| 参数 | 默认值 | 安全范围 | 影响 |
|------|--------|---------|------|
| `master_sfx_volume` 默认值 | 0.5 | 0.3–0.7 | 整体音效响度基准；过高会打扰工作，过低存在感消失 |
| `category_modifier` 泄漏音效 | 0.6 | 0.3–0.8 | 异世界声音的远近感；过高失去朦胧感，过低听不见 |
| `category_modifier` 动画帧音效 | 0.8 | 0.5–1.0 | 动作质感强弱；过低动作失去重量感 |
| `category_modifier` 持续底音 | 0.3 | 0.1–0.5 | Away 底音存在感；超过 0.5 容易造成长时间疲劳 |
| `category_modifier` 状态音效 | 1.0 | 0.7–1.0 | 状态切换的感知清晰度 |
| `category_modifier` 事件音效 | 1.0 | 0.8–1.0 | 出发/归来边界的感知强度；不建议调低 |
| 状态音效淡出时长 | 0.15s | 0.05–0.3s | 状态切换声音过渡流畅度；过长会造成声音堆叠感 |
| 静音淡出时长 | 0.1s | 0.05–0.2s | 静音响应速度；过长会有残响不适感 |
| Away 底音淡出时长 | 0.3s | 0.1–0.5s | 归来时底音消退的自然感 |
| `IDLE_SFX_COOLDOWN` | 60s | 30–120s | Idle 状态音触发频率；过短会感觉嘈杂，过长失去存在感 |
| `reverb_low` 混响湿度 | 0.4 | 0.2–0.7 | 泄漏音效的空间距离感；过高失真，过低空间感消失 |
| `reverb_low` 低通截止频率 | 800Hz | 400–1200Hz | 泄漏音效的模糊程度；过低声音太闷，过高太清晰 |
| `reverb_low` 混响衰减时间 | 1.2s | 0.5–3.0s | 泄漏音效尾音长度 |

## Acceptance Criteria

**信号响应**
- [ ] 收到 F2 `state_changed` 时，Fe5 在 50ms 内开始播放对应状态音效（若有映射）
- [ ] 收到 C1 `animation_event` 时，Fe5 在 16ms（1帧）内开始播放对应帧音效
- [ ] 收到 Fe2 `play_leak_audio(audio_id, "reverb_low")` 时，音效带混响+低通处理后播放
- [ ] 收到 C2 `outbound_triggered` 时播放出发事件音；收到 `return_completed` 时播放归来事件音

**并发控制**
- [ ] 状态音效并发上限为 1：新音到来时旧音在 0.15 秒内淡出，新音立即开始
- [ ] 动画帧音效并发上限为 2：超出时丢弃最短的一条，保留最新触发
- [ ] 泄漏音效并发上限为 1：新音直接替换旧音，不叠加
- [ ] 四类音效可同时并发播放，互不阻塞

**音量计算**
- [ ] 所有音效 `final_volume = asset_base_volume * master_sfx_volume * category_modifier`，结果符合公式预期
- [ ] `master_sfx_volume` 调整后，所有新触发音效立即反映新音量
- [ ] `category_modifier` 默认值：泄漏音 0.6，动画帧音 0.8，持续底音 0.3，状态/事件音 1.0

**静音与窗口**
- [ ] `set_mute(true)` 后：不播放任何新音效；已播音效在 0.1 秒内淡出
- [ ] `set_mute(false)` 后：恢复正常响应信号
- [ ] 收到 `window_hidden` 后：所有音效立即淡出，新触发静默丢弃

**Idle 冷却**
- [ ] Idle 状态音在同一次 Idle 持续中最多每 `IDLE_SFX_COOLDOWN`（60 秒）触发一次

**Away 底音**
- [ ] Away 进入时底音启动（若已配置资源）；`return_completed` 后底音在 0.3 秒内淡出
- [ ] 底音可被静音立刻关闭

**容错**
- [ ] 映射表无对应条目：静默跳过，记录调试日志，不崩溃
- [ ] 音效资源缺失：静默跳过，记录警告日志，不崩溃
- [ ] `play_leak_audio` 传入不支持的 effect：降级为无特效播放，记录警告日志
- [ ] 存档读取失败：使用默认值 `master_sfx_volume=0.5`，静音关闭，继续运行

**音效资源 MVP 要求**
- [ ] 至少覆盖以下映射：Idle、Attentive、Away（出发）、Returning（归来）各一条状态/事件音效
- [ ] 至少 3 条泄漏音效资源，支持 `reverb_low` 处理

## Open Questions

**1. Godot 4 音效处理能力确认**

`reverb_low` 预设需要混响 + 低通滤波。需确认 Godot 4 的 `AudioEffectReverb` + `AudioEffectLowPassFilter` 在常驻桌面场景下的 CPU 占用是否可接受。若性能不达标，降级方案为：仅用低通滤波，去掉混响。
- 负责人：开发者（技术原型验证阶段）
- 阻塞：泄漏音效实现

**2. 音效资源格式与压缩标准**

MVP 音效资源的格式（`.ogg` / `.wav`）、采样率、位深、压缩比尚未确定。常驻桌面应用对内存敏感，需设定资源预算上限。
- 建议：`.ogg`，44100Hz，16bit，单声道；单个音效文件 < 200KB
- 负责人：开发者 + 音频资源制作阶段确认

**3. 动画帧音效的精确同步机制**

C1 通过 `animation_event` 信号触发帧音效，但 Godot 4 的信号调度存在帧延迟。16ms 内响应的 AC 需要在技术实现阶段验证是否可达，若不可达则放宽为 32ms（2帧）。
- 负责人：开发者（C1 + Fe5 集成测试时验证）

**4. 多角色扩展时的音效隔离**

MVP 为单角色，Fe5 目前设计为全局单例。若后续支持多角色，需要考虑每个角色实例拥有独立的音效状态（如各自的 Away 底音）。当前设计不阻塞此扩展，但未显式支持。
- 目标层级：Alpha 层或更后期，届时评估是否需要重构
