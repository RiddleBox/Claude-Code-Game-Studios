# Fe2 — 泄漏内容系统（Leak Content System）

> **Status**: Approved
> **Author**: Claude Code Game Studios
> **Last Updated**: 2026-03-27
> **Implements Pillar**: 真实存在感（外出期间的好奇心钩子）/ 陪伴不打扰

## Overview

Fe2 泄漏内容系统负责在角色外出期间，周期性地将「另一个世界正在发生的事」以碎片化的方式呈现给玩家。它不直接叙述，而是通过视觉闪现、文字碎片、入口状态变化和微弱背景音四种形式制造暗示，唤起玩家对角色正在经历什么的好奇心。Fe2 订阅 C2 的 `away_content_tick` 信号，每次触发时从泄漏内容库中抽取一条内容展示；内容风格受 C5 性格变量影响（MVP 阶段 C5 未就绪时使用默认风格）。Fe2 是纯展示系统，不修改任何游戏状态，不产生叙事后果——它只是让外出的世界「可以被感知到」。

## Player Fantasy

玩家正在专心工作，屏幕角落突然一闪——那扇洞口的内壁亮了一下，像是有什么东西从里面经过。然后是一行若有若无的文字，出现在洞口边缘，一两秒后消散：「……它竟然敢这么说——」。玩家没有得到任何解释，故事没有继续，画面恢复平静。但好奇心已经种下：它遇到谁了？发生了什么？这种「刚好错过了什么」的感觉，比任何完整叙述都更吸引人——因为玩家的想象力会自动填满那些空白，让故事变得比实际更丰富。Fe2 的目标不是告诉玩家发生了什么，而是让玩家**想知道**。

## Detailed Design

### Core Rules

#### 1. 基本职责

1. Fe2 是纯展示系统，只负责在外出期间呈现泄漏内容和外出态指令响应，不修改游戏状态
2. Fe2 订阅 C2 的 `away_content_tick` 信号，每次触发时执行一次被动泄漏展示
3. Fe2 接收 C2 的 `away_command_response(response: AwayResponse)` 回调，执行主动响应展示
4. Fe2 维护一个**泄漏内容库**（数据驱动，JSON/资源文件），按类型和标签组织

#### 2. 泄漏内容的四种展示形式

5. 所有展示（被动泄漏和主动响应）均使用以下四种形式之一（或组合）：

| 形式 | 描述 | 持续时间 |
|------|------|----------|
| **文字泄漏** | 一行碎片文字出现在洞口边缘，淡入后停留，再淡出 | 3–5 秒 |
| **视觉闪现** | 洞口内部短暂亮起一个模糊画面（场景色调/轮廓），一闪而过 | 0.5–1.5 秒 |
| **入口状态变化** | 洞口光晕颜色变化、震动、粒子涌出等，无文字无画面 | 2–4 秒 |
| **微弱背景音** | 极低音量的环境音/情绪音效，加混响/低通滤波处理，可被环境噪音掩盖 | 2–6 秒 |

6. 形式权重（默认，被动泄漏）：文字 40%、入口状态变化 30%、视觉闪现 20%、背景音 10%
7. 背景音**始终以其他形式为主**，可叠加在视觉/文字泄漏上，不单独作为唯一形式出现

#### 3. 被动泄漏模式

8. 触发来源：C2 的 `away_content_tick` 信号
9. 感知语义：玩家「无意中」感知到角色在外面经历什么，角色并不知道被看到了
10. 内容从泄漏内容库中按规则抽取（见第5节）
11. 展示之间有最短间隔 `MIN_LEAK_INTERVAL`（默认 5 分钟），防止过于频繁
12. 展示中途窗口隐藏时，当次展示静默取消，不补播

#### 4. 主动响应模式（外出态指令响应）

13. 触发来源：C2 调用 `handle_away_command()` 后，通过 `away_command_response(response)` 通知 Fe2
14. 感知语义：角色「忙里偷闲」抽空回应玩家，能感觉到它当下很忙或身处特殊环境
15. 主动响应的展示特征与被动泄漏不同：
    - **文字风格**：不完整、有停顿、带环境干扰感（如「……好、好的，等我——」「收到，但现在有点——」）
    - **音频处理**：若有语音，加混响/低通滤波，体现「隔着空间传来」的朦胧感
    - **特效**：洞口短暂亮起后恢复暗场；或文字带干扰噪点效果
16. 主动响应**不受** `MIN_LEAK_INTERVAL` 限制，玩家发出指令即可触发
17. 主动响应优先级高于同时待播的被动泄漏；若正在展示被动泄漏，响应等待当前展示结束后播放

#### 5. 内容库结构

18. 每条泄漏内容是一个数据条目，包含：
    - `type`：形式类型（text / visual / portal / audio）
    - `content`：内容本体（文字字符串 / 视觉资源引用 / 音效引用）
    - `tags`：标签列表，如 `["combat", "funny", "danger", "peaceful"]`
    - `personality_affinity`：适合的性格类型（空列表 = 通用，`["curious"]` = 好奇型优先）
    - `event_context`：可选，绑定到特定 C4 事件线节点
    - `leak_mode`：`passive`（被动泄漏）或 `active`（主动响应）或 `both`

19. MVP 阶段内容库包含至少 **15 条**通用被动泄漏条目 + **5 条**通用主动响应条目，涵盖：
    - 被动：战斗/紧张、有趣/轻松、神秘/难以理解、日常/温馨各 3–4 条
    - 主动：「确认收到」类 2 条、「正忙但回应」类 2 条、「归来再说」类 1 条

#### 6. 内容抽取规则（被动泄漏）

20. 每次收到 `away_content_tick` 时，按以下规则抽取：
    - 若 C5 就绪：优先抽取与当前性格 `personality_affinity` 匹配的条目
    - 若 C5 未就绪（MVP）：从通用条目（`personality_affinity = []`）随机抽取
    - 排除上次展示的条目（避免重复）
    - 若当次外出已展示超过 `MAX_LEAKS_PER_OUTING` 条，停止抽取被动泄漏

### States and Transitions

Fe2 是无状态驱动的展示系统，内部维护一个简单的**展示队列状态**：

| 状态 | 描述 |
|------|------|
| **Idle**（待机） | 无展示进行中，等待触发 |
| **Displaying**（展示中） | 正在播放一条泄漏内容或主动响应 |
| **Queued**（排队） | 主动响应等待当前被动泄漏结束后播放 |

**状态转换**：

```
收到 away_content_tick（被动泄漏）：
  Idle → Displaying（立即播放）
  Displaying → 丢弃（被动泄漏不排队，错过即错过）

收到 away_command_response（主动响应）：
  Idle → Displaying（立即播放）
  Displaying → Queued（等待当前结束后播放）

展示完成：
  Displaying → Idle（队列为空）
  Displaying → Displaying（播放队列中的主动响应）

窗口 Hidden 时：
  Displaying → Idle（当前展示静默取消）
  Queued → Queued（保留主动响应，恢复可见后播放）

C2 状态离开 Away（归来）：
  任意 → Idle（清空所有被动泄漏队列）
  ※ 主动响应若在队列中，等归来演出结束后再播放
```

**关键规则**：
- 被动泄漏**不排队**——错过就错过，保持低打扰
- 主动响应**会排队**——玩家主动发出的指令必须有回应
- 同时最多 1 条主动响应在队列中

### Interactions with Other Systems

| 交互系统 | 数据流向 | Fe2 提供什么 | Fe2 需要什么 |
|---------|---------|------------|------------|
| **C2 外出-归来循环** | C2 → Fe2 | — | `away_content_tick` 信号（触发被动泄漏）；`away_command_response(response)` 回调（触发主动响应）；`return_completed` 信号（清空队列） |
| **C5 性格变量系统** | C5 → Fe2 | — | `get_personality_traits() -> Dictionary`（内容抽取时的性格过滤，MVP 阶段可选）|
| **C4 事件线系统** | C4 → Fe2 | — | `current_event_context: String`（可选，让泄漏内容与当前事件线节点关联）|
| **Fe5 声音系统** | Fe2 → Fe5 | `play_leak_audio(audio_id, effect: "reverb_low")` 请求 | — |
| **P1 主界面 UI** | Fe2 → P1 | 展示指令（文字内容、持续时间、特效类型）| 洞口/窗口的渲染区域位置 |
| **F1 桌面窗口系统** | F1 → Fe2 | — | `window_visibility_changed` 信号（窗口隐藏时取消展示）|

**接口约定**：
- Fe2 暴露方法 `receive_away_response(response: AwayResponse) -> void`，供 C2 调用
- Fe2 不暴露公开的内容触发接口——所有触发均来自信号，外部系统不直接控制展示
- P1 负责渲染，Fe2 只发送展示数据包（内容+时长+特效参数），不直接操作 UI 节点

## Formulas

Fe2 为纯展示系统，无游戏逻辑公式。以下为内容抽取的权重计算：

#### 展示形式权重抽取

```
被动泄漏形式选择（weighted random）：
  text:   40%
  portal: 30%
  visual: 20%
  audio:  10%  ← 仅作叠加，不单独触发

主动响应形式（固定优先级）：
  text + portal_glow（文字 + 洞口光晕）为默认组合
  若有语音资源：叠加 audio（reverb 处理）
```

#### 内容抽取优先级评分

当内容库有多个候选条目时，按以下分数排序取最高分：

```
score = base_score
      + personality_match_bonus   （C5 匹配时 +2，不匹配 -1，未接入 C5 时 +0）
      + event_match_bonus         （与当前 C4 事件上下文匹配时 +3）
      - recent_penalty            （上次展示过此条目：-10，两次内展示过：-5）
```

**变量定义**：

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `MIN_LEAK_INTERVAL` | 被动泄漏最短间隔（分钟） | 5 |
| `MAX_LEAKS_PER_OUTING` | 每次外出最多被动泄漏次数 | 6 |
| `TEXT_DISPLAY_DURATION` | 文字泄漏显示时长（秒） | 4 |
| `VISUAL_FLASH_DURATION` | 视觉闪现时长（秒） | 1 |
| `PORTAL_EFFECT_DURATION` | 入口状态变化时长（秒） | 3 |

## Edge Cases

**1. 内容库为空或所有条目被 recent_penalty 排除**
- 条件：MVP 阶段内容库条目不足，或本次外出已展示过所有条目
- 处理：跳过本次泄漏，不展示任何内容，记录调试日志；不报错

**2. 外出极短（10 分钟）时 `MIN_LEAK_INTERVAL` 导致无法触发泄漏**
- 条件：外出时长 10 分钟，`MIN_LEAK_INTERVAL` 5 分钟，最多只能触发 1–2 次
- 处理：属于正常情况，短暂外出本就应该少泄漏；不强制触发

**3. 主动响应在归来后仍在队列中**
- 条件：玩家发出指令后角色立即归来，响应还未播放
- 处理：归来演出结束后播放该响应，但以「归来态」的视觉表现（正常窗口内），而非外出态特效

**4. 同时收到多条 `away_command_response`（快速连续指令）**
- 条件：玩家快速连发多条指令，C2 快速回调多次
- 处理：只保留最新一条主动响应在队列中，丢弃之前未播放的响应；避免队列积压

**5. P1 尚未实现时 Fe2 发出展示指令**
- 条件：MVP 早期 P1 未就绪
- 处理：Fe2 发出展示信号无接收方，静默丢弃；Fe2 不因此崩溃

**6. 视觉闪现资源文件缺失**
- 条件：内容库条目引用的视觉资源不存在
- 处理：降级为入口状态变化（portal）效果，记录警告日志

**7. 外出期间 C5 性格突然变化（后续版本）**
- 条件：C5 性格值在外出中途变化
- 处理：下次抽取时使用新性格值，当次展示不中断

## Tuning Knobs

| 参数 | 当前值 | 安全范围 | 增大效果 | 减小效果 |
|------|--------|---------|---------|----------|
| `MIN_LEAK_INTERVAL` | 5 分钟 | 2 到 15 分钟 | 泄漏更稀疏，每次更有惊喜感 | 泄漏更频繁，可能让玩家感到打扰 |
| `MAX_LEAKS_PER_OUTING` | 6 条 | 2 到 10 条 | 外出期间信息更丰富 | 更克制，更神秘 |
| `TEXT_DISPLAY_DURATION` | 4 秒 | 2 到 8 秒 | 玩家有更多时间阅读 | 一闪而过，更神秘但可能错过 |
| `VISUAL_FLASH_DURATION` | 1 秒 | 0.3 到 2 秒 | 视觉内容更清晰可辨 | 更难看清，增加神秘感 |
| `PORTAL_EFFECT_DURATION` | 3 秒 | 1 到 6 秒 | 入口状态变化更明显 | 更短暂，更像偶发信号 |
| 形式权重（text/portal/visual/audio） | 40/30/20/10 | 各项 10–60% | 调整主要感知方式 | — |

**调优说明**：
- 核心体验目标：玩家在外出期间应偶尔「瞥到一眼」，而不是持续被通知
- 一次 30 分钟外出预期触发 3–5 次被动泄漏（MIN_LEAK_INTERVAL=5 时理论上最多 6 次，但受 away_content_tick 频率限制）
- 若玩家反馈「太多了」，优先增大 `MIN_LEAK_INTERVAL`；若反馈「感觉不到它在外面」，增加 `MAX_LEAKS_PER_OUTING` 或减小 `MIN_LEAK_INTERVAL`

## Visual/Audio Requirements

#### 视觉需求

**文字泄漏**：
- 字体：与游戏主字体一致，但使用半透明（opacity 60–80%）
- 位置：洞口边缘**外侧**，文字略微溢出窗口边缘，让玩家感知到「从里面漏出来」
- 连接感：文字带发光描边或扫光特效，颜色与当前洞口光晕色一致；可选在文字起点处绘制一条淡淡的光迹连回洞口边缘，强化「从洞口生长出来」的感觉
- 动画：淡入（0.3s）→ 停留（`TEXT_DISPLAY_DURATION`）→ 淡出（0.5s）
- 主动响应文字：额外加噪点/干扰线动画，体现「信号不稳定」感
- 具体特效字风格待美术方向确认后细化

**视觉闪现**：
- 内容：模糊的场景色块/轮廓，不呈现清晰画面——只是颜色和大致形状
- 动画：快速闪现（0.1s 亮起）→ 停留（`VISUAL_FLASH_DURATION`）→ 快速消散
- 模糊程度：高斯模糊半径 8–12px，玩家看不清细节

**入口状态变化**：
- 光晕颜色变化：按内容情绪映射（战斗/危险 → 红橙色，平静/有趣 → 蓝绿色，神秘 → 紫色）
- 粒子：少量从洞口内侧逸出，持续时间内自然消散
- 震动：仅用于「危险/激烈」类内容，幅度 2–4px，频率低，不让玩家误以为是系统故障

**主动响应特效**：
- 洞口光晕短暂亮起（更亮、更白），然后恢复暗场
- 文字带扫描线/干扰噪点动画

#### 音频需求

- 所有泄漏音效音量上限：**主音量的 20%**，确保不突兀
- 处理方式：高混响（reverb）+ 低通滤波（lowpass），模拟「隔空传来」的朦胧感
- 主动响应若有语音：额外加轻微失真（distortion 5–10%），体现次元传输感
- 音效类别：战斗（金属碰撞/脚步急促）、轻松（笑声片段/环境音）、神秘（不明声响）
- 具体音效内容由 Fe5 设计，Fe2 只定义触发参数

## UI Requirements

Fe2 不直接渲染 UI，向 P1 发送展示数据包，由 P1 负责实际渲染：

| 数据/接口 | 提供方 | 用途 |
|-----------|--------|------|
| `show_leak(data: LeakDisplayData)` | Fe2 → P1 | 触发一次泄漏展示，包含：内容类型、文字/资源引用、持续时间、特效参数、情绪标签 |
| `cancel_leak()` | Fe2 → P1 | 窗口隐藏时取消当前展示 |
| `get_portal_bounds() -> Rect2` | P1 → Fe2 | Fe2 查询洞口区域坐标，用于计算文字溢出位置 |

**LeakDisplayData 结构**（草稿）：
```
{
  type: String,          # "text" / "visual" / "portal" / "audio"
  content: String,       # 文字内容 或 资源路径
  duration: float,       # 展示持续时间（秒）
  glow_color: Color,     # 洞口光晕颜色
  fx_style: String,      # "normal" / "distorted"（主动响应用）
  emotion_tag: String,   # "combat" / "funny" / "mystery" / "peaceful"
  is_active_response: bool  # 区分被动泄漏和主动响应
}
```

**UI 约束**：
- 文字溢出位置不能遮挡玩家常用的交互区域（P1 设计时需预留）
- 所有泄漏展示结束后，界面完全恢复原状，不留残影

## Acceptance Criteria

**被动泄漏触发**
- [ ] 收到 `away_content_tick` 时，若当前 Idle 且未超过 `MAX_LEAKS_PER_OUTING`，Fe2 在 5 秒内完成展示初始化
- [ ] 两次被动泄漏实际展示间隔不短于 `MIN_LEAK_INTERVAL`（5 分钟）；间隔不足时静默丢弃，不报错
- [ ] 同一次外出被动泄漏次数达到 `MAX_LEAKS_PER_OUTING`（6 条）后，后续 `away_content_tick` 静默忽略
- [ ] 内容抽取时，上次展示过的条目 `recent_penalty` 扣 10 分，两次内展示过的扣 5 分，确保不连续重复同一条

**主动响应展示**
- [ ] 收到 `receive_away_response()` 时，若当前 Idle，立即开始播放
- [ ] 若当前正在展示被动泄漏，主动响应进入 Queued 状态，当前展示完成后立即播放
- [ ] 快速连发多条主动响应时，只保留最新一条在队列中，丢弃之前未播放的响应
- [ ] 主动响应不受 `MIN_LEAK_INTERVAL` 限制，可在任意时间触发

**展示形式与时长**
- [ ] 被动泄漏形式按权重随机选择：text 约 40%，portal 约 30%，visual 约 20%；audio 不单独出现，只叠加
- [ ] 文字泄漏总持续时间约为 `TEXT_DISPLAY_DURATION`（4 秒）：淡入 0.3s → 停留 → 淡出 0.5s
- [ ] 视觉闪现持续约 `VISUAL_FLASH_DURATION`（1 秒）：0.1s 快速亮起，停留后快速消散
- [ ] 入口状态变化持续约 `PORTAL_EFFECT_DURATION`（3 秒）
- [ ] 主动响应文字带干扰噪点/扫描线效果（`fx_style = "distorted"`），与被动泄漏视觉可区分

**窗口可见性处理**
- [ ] 展示进行中窗口切换为 Hidden：被动泄漏当次展示立即取消，不补播
- [ ] 展示进行中窗口切换为 Hidden：Queued 中的主动响应保留，窗口恢复 Visible 后继续播放
- [ ] 窗口恢复 Visible 后，若 Queued 中有主动响应，立即开始播放

**归来与状态重置**
- [ ] 收到 `return_completed` 信号后，Fe2 清空本次外出的被动泄漏计数和 `recent_penalty` 记录
- [ ] 归来时若主动响应在 Queued 中，归来演出结束后以归来态视觉（`fx_style = "normal"`）播放

**容错与降级**
- [ ] 内容库为空或所有候选条目因惩罚分被排除：静默跳过本次泄漏，输出调试日志，不崩溃
- [ ] 内容条目引用的视觉资源文件缺失：降级为 portal 效果展示，输出警告日志
- [ ] P1 未就绪时发出 `show_leak()` 信号无接收方：Fe2 不崩溃，静默丢弃

**内容库 MVP 要求**
- [ ] 内容库包含至少 15 条通用被动泄漏条目，覆盖战斗/有趣/神秘/日常各类别
- [ ] 内容库包含至少 5 条通用主动响应条目，覆盖「确认收到」「正忙回应」「归来再说」类型

## Open Questions

**1. 阶段化概率泄漏系统（v2 增强目标）**

当前 MVP 使用固定 `MIN_LEAK_INTERVAL` + `away_content_tick` 驱动被动泄漏。后续升级方向：
- **外出分阶段**：出发过渡 → 目的地/主事件 → 归来前（2-3 个阶段）
- **阶段概率曲线**：每阶段设置不同泄漏基础概率（出发低 → 主事件高 → 归来前中）
- **事件类型影响**：战斗/神秘/轻松/日常各自调整概率权重和内容池过滤
- **依赖**：C4 事件线提供事件类型；C2 `away_content_tick` 信号预留 `phase` 字段（见 C2 Open Questions）
- **目标层级**：垂直切片阶段（P1 层），C4 设计完成后接入
- **返工评估**：展示管道不需改动；选取逻辑替换为阶段概率曲线；数据条目补充阶段标签

**2. C5 性格对展示形式权重的影响**

当前性格仅影响内容抽取优先级（评分）。后续可让不同性格偏好不同展示形式：
- 好奇型 → 更多 visual flash
- 慵懒型 → 更多 text
- 活泼型 → 更多 portal 状态变化
- 依赖：C5 性格变量系统设计完成后接入

**3. 主动响应的视觉资源扩展**

MVP 阶段主动响应使用文字 + 洞口光晕组合。后续若为特定事件线节点定制语音，需与 Fe5 确认语音资源格式和混响处理参数。
